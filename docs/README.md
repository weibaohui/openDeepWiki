# 文档目录说明

本目录遵循项目文档编写规范，文档按照职责类型进行组织。

## 标准目录结构

```
docs/
├── requirements/     # 需求 & 实现总结（L1 开发需求）
│   ├── 001-仓库分支与版本管理-需求.md
│   ├── 001-仓库分支与版本管理-实现总结.md
│   └── .gitkeep
├── design/           # 设计 / 实现方案（编码前）
│   └── .gitkeep
├── testing/          # 测试设计文档
│   └── .gitkeep
└── decisions/        # 架构与关键决策（可选）
    ├── ADR-001-事件机制-决策.md
    ├── ADR-002-状态机与任务编排-决策.md
    └── .gitkeep
```

## 旧目录（保留用于向后兼容）

以下目录保留用于项目规范和总体文档：

- `开发规范/` - 前后端开发规范（前端规范、后端规范）
- `文档编写规范/` - 文档编写规范本身

## 文档命名规范

所有需求相关文档必须使用统一编号：
- `<编号>-<功能名称>-<文档类型>.md`

例如：
- `001-仓库分支与版本管理-需求.md`
- `001-仓库分支与版本管理-设计.md`
- `001-仓库分支与版本管理-测试.md`
- `001-仓库分支与版本管理-实现总结.md`

## 详细规范

请参考 `docs/文档编写规范/README.md` 了解完整的文档编写规范。
