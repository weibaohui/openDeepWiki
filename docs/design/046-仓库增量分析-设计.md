## 设计目标

1. 通过仓库详情页菜单按钮触发增量分析流程。
2. 以 repoID 为输入，读取数据库中仓库路径与基线 commit。
3. 在本地仓库执行 git pull 后进行增量差异分析。
4. 在后端打印变更内容，便于排查。

## 总体流程

1. 前端新增“仓库增量分析”按钮。
2. 点击按钮调用后端接口：`POST /api/repo/{id}/incremental-analysis`。
3. 后端查询仓库记录，获取 `local_path` 与 `commit`。
4. 后端在 `local_path` 执行 `git pull` 并校验成功。
5. 后端调用 `FormatIncrementalChangesForAI(repoPath, baseCommit)` 生成变更摘要。
6. 后端使用 `klog.V(6)` 中文日志输出变更内容。
7. 接口返回成功或错误信息。

## 关键设计点

### 接口设计

- 路径：`POST /api/repo/{id}/incremental-analysis`
- 入参：路径参数 `id`（仓库 ID）
- 返回：
  - 成功：`{"success": true}`
  - 失败：`{"success": false, "error": "xxx"}`

### 数据来源

- 数据库表：仓库记录表
- 字段需求：
  - 本地路径：用于 git pull 与差异分析
  - 基线 commit：用于增量分析 base commit

### 业务日志

- 使用 `klog.V(6)` 输出：
  - repoID、仓库路径、base commit
  - git pull 执行结果
  - 增量变更摘要

### 工具支持

- 增量任务规划 Agent 需要 `read_doc` 工具读取文档全文，用于对比增量语义与现有文档内容。
- 数据来源为 DocumentRepository，按 doc_id 获取文档内容。

## 影响范围

- 前端：仓库详情页面按钮与调用逻辑
- 后端：新增接口、调用 git 增量分析函数
- 路由：新增仓库相关接口路由

## 异常处理

- 仓库不存在：返回错误并记录日志
- 本地路径不存在：返回错误并记录日志
- git pull 失败：返回错误并记录日志
- 增量分析失败：返回错误并记录日志
