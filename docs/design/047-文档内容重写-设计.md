# 1. 设计目标

在现有 Writer 体系中新增 DocRewriter，实现基于任务 Outline 指引的文档内容重写能力，并保持与 TitleRewriter 相同的调用方式与错误处理规范。

# 2. 核心流程

1. 根据任务ID读取任务信息，获取 DocID 与 Outline 指引。
2. 从文档库读取原始文档内容。
3. 调用 DocRewriter Agent，将原文内容与指引合并为输入提示词。
4. 仅接受 Agent 返回的 Markdown 文本作为新内容。
5. 若新旧内容不同，则保存文档内容更新。

# 3. 关键设计点

## 3.1 Writer 接入

- 新增 DocRewriter 实现，满足 Writer 接口。
- 在服务启动时完成注册，确保任务调度可匹配 WriterName。

## 3.2 Agent 设计

- 新增 doc_rewriter Agent 配置文件，约束输出为纯 Markdown 文本。
- 允许依据更新内容与替换内容执行替换、补充或删除。

# 4. 影响范围

- backend/internal/domain/writers/
- backend/cmd/server/main.go
- backend/agents/

# 5. 风险与兼容性

- 不变更数据库结构，保持现有文档保存逻辑。
- 当 Agent 输出为空时返回错误，避免保存空文档。
