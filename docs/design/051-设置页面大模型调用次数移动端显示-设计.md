## 1. 设计概述

本文档描述如何修复设置页面 API Key 列表中 `Request Count` 列在移动端不显示的问题。

---

## 2. 问题分析

### 2.1 根本原因

在 `frontend/src/components/settings/APIKeyList.tsx` 文件中，`request_count` 列定义了 `responsive: ['lg']` 属性：

```typescript
{
    title: t('apiKey.request_count', 'Request Count'),
    dataIndex: 'request_count',
    key: 'request_count',
    render: (text: number) => text ? text.toString() : '-',
    responsive: ['lg']  // ← 这行导致了移动端不显示
}
```

根据 Ant Design 的响应式设计：
- `lg` 对应屏幕宽度 >= 992px（PC 端）
- 当屏幕宽度 < 992px（移动端）时，该列会被隐藏

### 2.2 影响范围

- **影响文件**：`frontend/src/components/settings/APIKeyList.tsx`
- **影响组件**：`APIKeyList`
- **影响列**：`request_count` 列
- **影响页面**：设置页面 -> API Key Management 标签页

---

## 3. 设计方案

### 3.1 解决方案

移除 `request_count` 列的 `responsive: ['lg']` 属性，使该列在所有屏幕尺寸下都显示。

### 3.2 变更说明

**文件**：`frontend/src/components/settings/APIKeyList.tsx`

**位置**：第 168-173 行

**变更内容**：删除 `responsive: ['lg']` 属性

**变更后代码**：
```typescript
{
    title: t('apiKey.request_count', 'Request Count'),
    dataIndex: 'request_count',
    key: 'request_count',
    render: (text: number) => text ? text.toString() : '-'
}
```

### 3.3 设计决策

1. **为何移除响应式限制**：用户明确要求 PC 和手机端都能看到调用次数
2. **为何不使用其他响应式断点**：不限制显示，确保所有设备都能看到该列
3. **为何不改变列顺序**：保持现有的列顺序，只修改显示逻辑

### 4.2 兼容性考虑

- **向后兼容**：移除响应式限制后，PC 端的显示效果与之前一致
- **移动端体验**：由于表格本身已支持横向滚动（`scroll={{ x: 800 }}`），新增的列不会破坏布局

### 5.1 测试场景

| 场景 | 预期结果 |
|------|---------|
| PC 端访问设置页面 | 能看到 `Request Count` 列 |

## 6. 风险评估

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| 移动端表格过宽，横向滚动体验差 | 低 | 表格已配置 `scroll={{ x: 800 }}`，支持横向滚动 |
| 其他列受影响 | 无 | 仅修改 `request_count` 列的配置 |
| 数据显示错误 | 无 | 仅修改显示逻辑，不涉及数据处理 |
