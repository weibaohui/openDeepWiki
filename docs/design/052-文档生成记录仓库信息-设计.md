# 1. 设计概述

本设计文档描述文档生成时记录仓库信息的实现方案。该功能旨在为每个文档记录生成时使用的代码仓库分支和 commit id，并在文档查看页面显示这些信息，提高文档的可追溯性和可信度。

核心设计思路：
1. 在 Document 模型中添加 clone_branch 和 clone_commit_id 字段
2. 在文档保存时，从关联的 Repository 获取这些信息并保存
3. 在前端文档查看页面底部显示仓库信息块，样式参考 TokenUsage 显示块

# 2. 数据模型设计

## 2.1 Document 模型修改

在现有的 Document 模型中添加两个新字段：

### 模型定义

```go
// backend/internal/model/models.go

type Document struct {
    ID             uint      `json:"id" gorm:"primaryKey"`
    RepositoryID   uint      `json:"repository_id" gorm:"index;"`
    TaskID         uint      `json:"task_id" gorm:"index"`
    Title          string    `json:"title" gorm:"size:255;"`
    Filename       string    `json:"filename" gorm:"size:255;"`
    Content        string    `json:"content" gorm:"type:text"`
    SortOrder      int       `json:"sort_order" gorm:"default:0"`
    Version        int       `json:"version" gorm:"default:1;index"`
    IsLatest       bool      `json:"is_latest" gorm:"index"`
    ReplacedBy     uint      `json:"replaced_by" gorm:"index;"`
    CloneBranch    string    `json:"clone_branch" gorm:"size:255"`      // 新增：生成文档时的分支
    CloneCommitID  string    `json:"clone_commit_id" gorm:"size:100"`    // 新增：生成文档时的 commit id
    CreatedAt      time.Time `json:"created_at"`
    UpdatedAt      time.Time `json:"updated_at"`
}
```

### 字段说明

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| CloneBranch | string | 生成文档时的分支名称 | SIZE:255 |
| CloneCommitID | string | 生成文档时的 commit id | SIZE:100 |

## 2.2 数据库迁移

### 自动迁移

GORM 的 AutoMigrate 会自动创建新字段，无需手动编写迁移脚本。

### 历史数据处理

对于已存在的 Document 记录，可以在系统升级时通过脚本填充默认值（如果关联的 Repository 有 clone_branch 和 clone_commit_id）。

```sql
-- 示例：历史数据填充（可选）
UPDATE documents d
SET clone_branch = r.clone_branch,
    clone_commit_id = r.clone_commit_id
FROM repositories r
WHERE d.repository_id = r.id
  AND (d.clone_branch = '' OR d.clone_branch IS NULL);
```

# 3. 后端设计

## 3.1 Repository 层修改

### 文件：backend/internal/repository/doc_repo.go

修改 `Create` 和 `CreateVersioned` 方法，在保存文档时填充仓库信息：

```go
// Create 创建文档
func (r *documentRepository) Create(doc *model.Document) error {
    // 从关联的 Repository 获取 clone_branch 和 clone_commit_id
    var repo model.Repository
    if err := r.db.First(&repo, doc.RepositoryID).Error; err == nil {
        doc.CloneBranch = repo.CloneBranch
        doc.CloneCommitID = repo.CloneCommit
    }
    return r.db.Create(doc).Error
}

// CreateVersioned 创建版本化文档
func (r *documentRepository) CreateVersioned(doc *model.Document) error {
    return r.db.Transaction(func(tx *gorm.DB) error {
        var maxVersion sql.NullInt64
        if err := tx.Model(&model.Document{}).
            Where("task_id = ?", doc.TaskID).
            Select("MAX(version)").
            Scan(&maxVersion).Error; err != nil {
            return err
        }

        nextVersion := 1
        if maxVersion.Valid {
            nextVersion = int(maxVersion.Int64) + 1
        }

        if err := tx.Model(&model.Document{}).
            Where("task_id = ? AND is_latest = ?", doc.TaskID, true).
            Updates(map[string]interface{}{
                "is_latest":  false,
                "updated_at": time.Now(),
            }).Error; err != nil {
            return err
        }

        // 从关联的 Repository 获取 clone_branch 和 clone_commit_id
        var repo model.Repository
        if err := tx.First(&repo, doc.RepositoryID).Error; err == nil {
            doc.CloneBranch = repo.CloneBranch
            doc.CloneCommitID = repo.CloneCommit
        }

        doc.Version = nextVersion
        doc.IsLatest = true
        return tx.Create(doc).Error
    })
}
```

# 5. 安全考虑

1. **敏感信息**：clone_branch 和 clone_commit_id 是公开的仓库元信息，不涉及敏感数据
2. **数据验证**：从 Repository 表获取数据，无需额外验证
3. **防 XSS**：前端使用 React 自动转义，显示 commit id 时使用 Ant Design 的 Text code 组件

# 6. 性能考虑

1. **数据库查询**：在文档保存时需要查询 Repository 表，增加一次查询
   - 影响：文档生成过程本来就需要查询 Repository，影响较小
   - 优化：可以在事务中使用预加载一次性获取

2. **前端渲染**：仅在有仓库信息时渲染显示块，不影响性能
