## 1. 设计目标

### 1.1 要解决的问题

1. **缺乏实时交互**：现有系统基于离线任务模式，用户无法与AI进行实时对话
2. **思考过程黑盒**：用户无法了解AI如何得出答案，降低了可信度
3. **无法干预控制**：用户在AI回答过程中无法停止或修正方向
4. **对话无法追溯**：没有保存对话历史，无法回顾之前的问答

### 1.2 功能边界

**包含：**
- 基于代码仓库的实时问答对话
- 思考过程和工具调用的可视化展示
- 会话的创建、实时保存、加载
- 回答过程中的停止功能
- 追加输入和重新作答

**不包含（后续迭代）：**
- 对话分支管理
- 对话分享
- 语音输入输出
- 上下文压缩优化

### 3.3 Agent执行器设计

#### 3.3.1 ChatAgentExecutor

```go
// ChatAgentExecutor 封装Eino ADK的Agent执行，支持流式输出和中断
type ChatAgentExecutor struct {
    agent        adk.Agent
    sessionID    string
    repoID       uint
    eventHandler ChatEventHandler
    repo         *repository.Repository  // 代码仓库信息
}

// ExecuteResult 执行结果
type ExecuteResult struct {
    Content   string
    TokenUsed int
    ToolCalls []ChatToolCall
    Err       error
}

// Execute 执行Agent，支持中断
func (e *ChatAgentExecutor) Execute(ctx context.Context, messages []ChatMessage, streamChan chan<- ChatEvent) error {
    // 1. 构建ADK消息
    adkMessages := e.convertToADKMessages(messages)

    // 2. 创建Runner
    runner := adk.NewRunner(ctx, adk.RunnerConfig{
        Agent: e.agent,
    })

    // 3. 启动执行迭代器
    iter := runner.Run(ctx, adkMessages)

    // 4. 遍历事件
    for {
        select {
        case <-ctx.Done():
            // 收到取消信号，停止执行
            streamChan <- ChatEvent{Type: EventTypeStopped}
            return ctx.Err()

        default:
            event, ok := iter.Next()
            if !ok {
                return nil
            }

            // 处理各种事件类型
            if err := e.handleEvent(event, streamChan); err != nil {
                return err
            }
        }
    }
}

// handleEvent 处理Agent事件并转换为ChatEvent
func (e *ChatAgentExecutor) handleEvent(event adk.Event, streamChan chan<- ChatEvent) error {
    switch event.Type {
    case adk.EventTypeThinking:
        streamChan <- ChatEvent{
            Type:    EventTypeThinkingStart,
            Content: event.Thinking,
        }

    case adk.EventTypeToolCall:
        streamChan <- ChatEvent{
            Type:     EventTypeToolCall,
            ToolCall: e.convertToolCall(event.ToolCall),
        }

    case adk.EventTypeToolResult:
        streamChan <- ChatEvent{
            Type:       EventTypeToolResult,
            ToolResult: e.convertToolResult(event.ToolResult),
        }

    case adk.EventTypeContent:
        streamChan <- ChatEvent{
            Type:    EventTypeContentDelta,
            Content: event.Content,
        }

    case adk.EventTypeFinish:
        streamChan <- ChatEvent{
            Type:      EventTypeAssistantEnd,
            TokenUsed: event.TokenUsage.TotalTokens,
        }
    }

    return nil
}
```

#### 3.3.2 中断机制

```go
// StopController 停止控制器
type StopController struct {
    cancelFunc context.CancelFunc
    stopped    chan struct{}
}

// NewStopController 创建停止控制器
func NewStopController() *StopController {
    return &StopController{
        stopped: make(chan struct{}),
    }
}

// Start 开始执行，返回可取消的context
func (s *StopController) Start() context.Context {
    ctx, cancel := context.WithCancel(context.Background())
    s.cancelFunc = cancel
    return ctx
}

// Stop 发送停止信号
func (s *StopController) Stop() {
    if s.cancelFunc != nil {
        s.cancelFunc()
        close(s.stopped)
    }
}

// IsStopped 检查是否已停止
func (s *StopController) IsStopped() bool {
    select {
    case <-s.stopped:
        return true
    default:
        return false
    }
}
```

### 3.4 数据持久化设计

#### 3.4.1 实时保存策略

```go
// MessagePersistence 消息持久化
type MessagePersistence struct {
    sessionRepo ChatSessionRepository
    messageRepo ChatMessageRepository
    toolRepo    ChatToolCallRepository
    // 异步保存通道
    saveChan chan SaveTask
}

// SaveTask 保存任务
type SaveTask struct {
    Type    string  // message, tool_call, tool_result
    Data    interface{}
    Done    chan error
}

// SaveUserMessage 保存用户消息（同步，立即保存）
func (p *MessagePersistence) SaveUserMessage(ctx context.Context, msg *ChatMessage) error {
    return p.messageRepo.Create(ctx, msg)
}

// SaveAssistantMessage 创建AI消息记录（初始状态为streaming）
func (p *MessagePersistence) SaveAssistantMessage(ctx context.Context, sessionID string) (*ChatMessage, error) {
    msg := &ChatMessage{
        MessageID:   generateID("msg"),
        SessionID:   sessionID,
        Role:        "assistant",
        Status:      "streaming",
        ContentType: "text",
        CreatedAt:   time.Now(),
    }
    return msg, p.messageRepo.Create(ctx, msg)
}

// UpdateMessageContent 流式更新消息内容
func (p *MessagePersistence) UpdateMessageContent(ctx context.Context, messageID string, content string) error {
    return p.messageRepo.UpdateContent(ctx, messageID, content)
}

// FinalizeMessage 完成消息保存
func (p *MessagePersistence) FinalizeMessage(ctx context.Context, messageID string, tokenUsed int, status string) error {
    return p.messageRepo.Finalize(ctx, messageID, tokenUsed, status, time.Now())
}

// SaveToolCall 保存工具调用（异步）
func (p *MessagePersistence) SaveToolCall(toolCall *ChatToolCall) {
    p.saveChan <- SaveTask{
        Type: "tool_call",
        Data: toolCall,
        Done: make(chan error, 1),
    }
}

// UpdateToolResult 更新工具调用结果（异步）
func (p *MessagePersistence) UpdateToolResult(toolCallID string, result string, durationMs int) {
    p.saveChan <- SaveTask{
        Type: "tool_result",
        Data: &ToolResultData{
            ToolCallID: toolCallID,
            Result:     result,
            DurationMs: durationMs,
        },
        Done: make(chan error, 1),
    }
}

// StartWorker 启动异步保存工作协程
func (p *MessagePersistence) StartWorker() {
    go func() {
        for task := range p.saveChan {
            var err error
            switch task.Type {
            case "tool_call":
                err = p.toolRepo.Create(context.Background(), task.Data.(*ChatToolCall))
            case "tool_result":
                data := task.Data.(*ToolResultData)
                err = p.toolRepo.UpdateResult(context.Background(), data.ToolCallID, data.Result, data.DurationMs)
            }
            task.Done <- err
        }
    }()
}
```

## 5. 错误处理设计

### 5.1 错误类型

| 错误类型 | 场景 | 处理方式 |
|----------|------|----------|
| ConnectionError | WebSocket连接断开 | 自动重连3次，失败提示用户手动刷新 |
| AuthError | 鉴权失败 | 提示重新登录，重定向到登录页 |
| ValidationError | 消息格式错误 | 前端校验，阻止发送 |
| ModelError | 模型调用失败 | 显示友好提示，保留已生成内容 |
| TimeoutError | 响应超时 | 显示超时提示，提供重试按钮 |
| RateLimitError | 频率限制 | 提示稍后再试，显示倒计时 |

### 5.2 错误响应格式

```typescript
interface ErrorMessage {
  type: 'error';
  error: {
    code: string;           // 错误码
    message: string;        // 错误消息
    details?: unknown;      // 详细信息
    retryable: boolean;     // 是否可重试
  };
}

// 错误码定义
enum ErrorCode {
  CONNECTION_ERROR = 'CONNECTION_ERROR',
  AUTH_ERROR = 'AUTH_ERROR',
  VALIDATION_ERROR = 'VALIDATION_ERROR',
  MODEL_ERROR = 'MODEL_ERROR',
  TIMEOUT_ERROR = 'TIMEOUT_ERROR',
  RATE_LIMIT_ERROR = 'RATE_LIMIT_ERROR',
  INTERNAL_ERROR = 'INTERNAL_ERROR',
}
```

## 6. 安全设计

### 6.1 鉴权流程

```
WebSocket连接建立
       │
       ▼
┌─────────────┐

       │

       │
       ▼
┌─────────────┐
│  验证会话归属 │     ┌─────────┐
```

### 6.2 防护措施

1. **消息长度限制**
   - 单条消息最大 40000 字符
   - 超过则前端阻止发送并提示

2. **消息数量限制**
   - 单个会话最大 1000 条消息
   - 达到限制后提示新建会话

3. **频率限制**
   - 单个用户最多 3 个并发 WebSocket 连接
   - 单个会话 10 秒内只能发送 1 条消息

4. **XSS防护**
   - 用户输入转义显示
   - Markdown渲染时过滤危险标签
