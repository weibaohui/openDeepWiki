## 一、需求背景

根据 [056-向量搜索功能-需求.md](../requirements/056-向量搜索功能-需求.md) 和 [056-向量搜索功能-实现总结.md](../requirements/056-向量搜索功能-实现总结.md)，需要开发前端管理页面，用于管理文档和仓库的向量化索引。

## 二、功能目标

1. **向量状态总览**：显示系统整体向量化状态
2. **仓库向量管理**：按仓库查看和管理向量化状态
3. **文档向量管理**：查看单个文档的向量状态，支持手动触发
4. **向量任务监控**：查看向量生成任务的状态

## 三、界面设计

### 3.3 功能说明

#### 3.3.1 状态统计卡片

| 字段 | 说明 |
|------|------|
| 总文档数 | 系统中所有文档数量 |
| 已向量化 | 已生成向量的文档数量 |
| 待处理 | 等待生成向量的任务数量 |
| 处理中 | 正在生成向量的任务数量 |
| 失败 | 生成向量失败的任务数量 |

#### 3.3.2 仓库向量化状态表格

| 字段 | 说明 |
|------|------|
| 仓库名称 | 仓库的名称 |
| 文档数 | 仓库下的文档总数 |
| 已向量化 | 已生成向量的文档数 |
| 状态 | 未开始 / 部分完成 / 完成 |
| 操作 | 生成向量、重新生成 |

#### 3.3.3 向量任务列表

| 字段 | 说明 |
|------|------|
| 文档ID | 文档的唯一标识 |
| 仓库名称 | 所属仓库 |
| 状态 | pending / processing / completed / failed |
| 创建时间 | 任务创建时间 |
| 错误信息 | 失败时的错误信息 |

## 四、API 接口

### 4.1 已有接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/vectors/status | 获取向量生成状态 |
| POST | /api/documents/:id/vector/generate | 为文档生成向量 |
| POST | /api/repositories/:id/vectors/generate | 批量为仓库生成向量 |
| POST | /api/documents/:id/vector/regenerate | 重新生成文档向量 |

### 4.2 新增接口
