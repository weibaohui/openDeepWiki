## 设计目标

将新会话创建时机从"点击创建"延迟到"发送第一条消息"，避免产生空会话记录。

## 核心设计

### 1. 临时会话标识

引入 `isTemporary` 标记区分临时会话和真实会话：

```typescript
// ChatSession 类型扩展
interface ChatSession {
  // ... 原有字段
  isTemporary?: boolean;  // 标记是否为临时会话
}
```

临时会话的特征：
- `isTemporary: true`
- `session_id` 为临时生成的客户端 ID（如 `temp_${timestamp}`）
- 不调用后端 API 创建
- 仅在本地状态中存在的会话

### 2. 状态管理调整

在 `useChat` hook 中增加临时会话管理：

```typescript
// 创建临时会话（纯本地操作）
const createTemporarySession = useCallback(() => {
  const tempSession: ChatSession = {
    id: Date.now(),
    session_id: `temp_${Date.now()}`,  // 临时 ID
    repo_id: repoId,
    title: '新对话',
    status: 'active',
    isTemporary: true,  // 标记为临时
    created_at: new Date().toISOString(),
    updated_at: new Date().toISOString(),
  };

  currentSessionIdRef.current = tempSession.session_id;

  setState((prev) => ({
    ...prev,
    currentSession: tempSession,
    sessions: [tempSession, ...prev.sessions],
    messages: [],
  }));
}, [repoId]);
```

### 3. 发送消息流程改造

修改 `sendMessage` 函数，在发送前检查并处理临时会话：

```typescript
const sendMessage = useCallback(async (content: string) => {
  if (!content.trim()) return;

  const currentSession = state.currentSession;
  if (!currentSession) {
    if (onError) onError('没有活跃的会话');
    return;
  }

  // 如果是临时会话，先创建真实会话
  if (currentSession.isTemporary) {
    try {
      const response = await chatApi.createSession(repoId);
      const realSession: ChatSession = {
        id: Date.now(),
        session_id: response.data.session_id,
        repo_id: repoId,
        title: content.trim().slice(0, 20) || '新对话',  // 使用消息前20字作为标题
        status: 'active',
        created_at: response.data.created_at,
        updated_at: response.data.created_at,
      };

      // 更新当前会话和会话列表
      currentSessionIdRef.current = realSession.session_id;
      setState((prev) => ({
        ...prev,
        currentSession: realSession,
        sessions: prev.sessions.map((s) =>
          s.session_id === currentSession.session_id ? realSession : s
        ),
      }));

      // 建立 WebSocket 连接
      disconnect();
      connect();

      // 等待连接建立后发送消息
      // ... 发送消息逻辑
    } catch (err) {
      if (onError) onError('创建会话失败');
      return;
    }
  }

  // ... 原有发送消息逻辑
}, [state.currentSession, repoId, connect, disconnect, onError]);
```

### 4. 连接管理调整

临时会话不应该建立 WebSocket 连接：

```typescript
const connect = useCallback(() => {
  const targetSessionId = currentSessionIdRef.current;

  // 临时会话不建立连接
  if (!targetSessionId || targetSessionId.startsWith('temp_')) {
    console.log('[WebSocket] Temporary session, skipping connection');
    return;
  }

  // ... 原有连接逻辑
}, [repoId, onError, updateState]);
```

### 5. 会话切换处理

切换到其他会话时，清理临时会话：

```typescript
const loadSession = useCallback(async (targetSessionId: string) => {
  // 如果当前是临时会话，从列表中移除
  setState((prev) => {
    const current = prev.currentSession;
    if (current?.isTemporary) {
      return {
        ...prev,
        sessions: prev.sessions.filter((s) => s.session_id !== current.session_id),
      };
    }
    return prev;
  });

  // ... 原有加载逻辑
}, [repoId, connect, disconnect, updateState, onError]);
```

### 6. 创建真实会话流程

创建真实会话的逻辑内联在 `sendMessage` 中（避免函数顺序依赖问题）：

```typescript
if (currentSession.isTemporary) {
  try {
    // 内联创建真实会话
```

## 异常处理

1. **创建会话 API 失败**：提示用户"创建会话失败，请重试"，保留临时会话状态
2. **WebSocket 连接失败**：同现有处理，自动重连
3. **发送消息失败**：同现有处理，显示错误提示

## 测试要点

1. 点击创建新对话，检查网络面板，确认没有调用 `/sessions` POST API
2. 在临时会话中输入消息并发送，检查是否先调用创建会话 API
3. 验证会话列表中临时会话在发送成功后转换为真实会话
4. 验证切换到其他会话后临时会话被清理
5. 验证页面刷新后临时会话消失
