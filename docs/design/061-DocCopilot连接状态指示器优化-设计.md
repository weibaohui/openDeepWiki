# DocCopilot 连接状态指示器优化设计

## 变更记录表

| 版本 | 日期 | 变更内容 | 作者 |
|------|------|----------|------|
| v1.0 | 2025-03-04 | 初始设计文档创建 | AI Assistant |

---

## 1. 需求背景

当前 DocCopilot 组件在连接断开时，会显示明显的文字提示"连接已断开"和"重新连接"按钮，占用较多界面空间，影响用户体验。由于采用延迟创建会话策略，首次进入页面时未建立连接是正常状态，不应显示明显的警告提示。

## 2. 设计目标

将连接状态指示器改为简洁的图标形式：
- 未连接状态：小灰点图标
- 已连接状态：小绿点图标
- 指示器位置放在标题栏，简单隐蔽

## 3. 详细设计

### 3.1 界面变更

**Before:**
```
┌─────────────────────────────────────┐
│  AI 助手              [+][放大][×]  │
├─────────────────────────────────────┤
│  ┌─────────────────────────────┐   │  <-- 占据整行
│  │ 连接已断开    [重新连接]    │   │
│  └─────────────────────────────┘   │
├─────────────────────────────────────┤
│                                     │
│         欢迎界面                    │
│                                     │
├─────────────────────────────────────┤
│ [输入框...                    ][发送]│
└─────────────────────────────────────┘
```

**After:**
```
┌─────────────────────────────────────┐
│ ● AI 助手             [+][放大][×]  │  <-- 灰点/绿点在标题前
├─────────────────────────────────────┤
│                                     │  <-- 移除整行提示
│         欢迎界面                    │
│                                     │
├─────────────────────────────────────┤
│ [输入框...                    ][发送]│
└─────────────────────────────────────┘
```

### 3.2 颜色定义

| 状态 | 颜色 | Token |
|------|------|-------|
| 未连接 (disconnected) | 灰色 | `token.colorTextDisabled` 或 `#999` |
| 已连接 (connected) | 绿色 | `#52c41a` (Ant Design 成功绿) |
| 重连中 (reconnecting) | 橙色/黄色 | `token.colorWarning` |

### 3.3 交互逻辑

- 点击状态图标可触发重新连接（可选）
- 保留原有自动连接逻辑：输入框聚焦时自动建立连接

### 3.4 代码变更范围

**文件:** `frontend/src/pages/DocCopilot.tsx`

**主要变更:**
1. 移除原有的 `connectionAlert` 样式区块（第 414-433 行）
2. 在 `headerTitle` 中添加状态圆点图标
3. 使用 Tooltip 显示当前连接状态的文字说明（可选增强）

### 3.5 状态指示器实现

```typescript
// 状态圆点样式
const statusDotStyle = (status: string): React.CSSProperties => ({
  width: 8,
  height: 8,
  borderRadius: '50%',
  backgroundColor: status === 'connected' ? '#52c41a' : '#999',
  display: 'inline-block',
  marginRight: 8,
});
```

## 4. 影响分析

- **无 API 变更**
- **无状态管理变更**
- **仅 UI 表现层调整**
- **对用户操作习惯无影响**

## 5. 测试要点

1. 首次进入页面时显示灰点
2. 点击输入框聚焦后，连接建立后变为绿点
3. 网络异常时恢复为灰点
4. 重连过程中可显示黄点（可选）
