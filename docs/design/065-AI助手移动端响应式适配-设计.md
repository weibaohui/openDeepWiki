## 1. 设计目标

实现 AI 助手在移动端的响应式适配，使其在小屏设备上提供更好的用户体验。

## 2. 架构设计

### 2.1 组件关系

```
DocViewer (容器组件)
├── 左侧文档列表 (Sider/Drawer)
├── 中间文档内容区域 (Layout)
│   └── Header (标题栏)
│       └── AI 按钮 (条件渲染: !copilotOpen)
└── 右侧 AI 助手区域 (条件渲染: copilotOpen)
    └── DocCopilot
        ├── Sidebar (会话列表)
        └── ChatArea (聊天区域)
```

### 2.2 状态管理

| 状态 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| copilotOpen | boolean | AI 助手开关状态 | false |
| copilotExpanded | boolean | AI 助手展开状态 | false |
| screens.md | boolean \| undefined | 响应式屏幕检测 | undefined |

## 3. 详细设计

### 3.1 DocViewer 响应式逻辑

#### 3.1.1 状态初始化
```typescript
// 默认关闭 AI 助手
const [copilotOpen, setCopilotOpen] = useState(false);
const [copilotExpanded, setCopilotExpanded] = useState(false);
```

#### 3.1.2 点击处理
```typescript
onClick={() => {
    setCopilotOpen(true);
    // 小屏下自动展开（全屏模式）
    if (!screens.md) {
        setCopilotExpanded(true);
    }
}}
```

#### 3.1.3 AI 助手容器样式
```typescript
<div style={{
    display: 'flex',
    flexDirection: 'column',
    height: '100vh',
    // 小屏下始终全屏
    flex: (copilotExpanded || !screens.md) ? 1 : 'unset',
    width: (copilotExpanded || !screens.md) ? '100%' : undefined,
    // 小屏下绝对定位覆盖
    position: !screens.md ? 'absolute' : undefined,
    top: !screens.md ? 0 : undefined,
    left: !screens.md ? 0 : undefined,
    zIndex: !screens.md ? 100 : undefined,
}}>
```

### 3.2 DocCopilot 样式适配

#### 3.2.1 紧凑模式（compactMode）
```css
.compactMode {
    flex-direction: column;
    width: 380px;
    @media (max-width: 768px) {
        width: 100%;
    }
}
```

#### 3.2.2 展开模式（expandedMode）
```css
.expandedMode {
    flex-direction: row;
    flex: 1;
    width: 100%;
}
```

### 3.3 状态通知优化

**问题**：DocCopilot 组件挂载时立即调用 `onExpandChange(false)`，导致 AI 助手被意外关闭。

**解决方案**：
```typescript
const [isExpanded, setIsExpanded] = useState(false);
const prevExpandedRef = useRef(isExpanded);

useEffect(() => {
    // 只在状态真正变化时才通知父组件
    if (prevExpandedRef.current !== isExpanded) {
        prevExpandedRef.current = isExpanded;
        onExpandChange?.(isExpanded);
    }
}, [isExpanded, onExpandChange]);
```

## 4. 响应式断点

| 断点 | 宽度 | AI 助手行为 |
|------|------|-------------|
| md | >= 768px | 默认打开，可展开/收缩 |
| 小屏 | < 768px | 默认关闭，点击后全屏 |

## 6. 关键实现点

### 6.1 状态抖动问题
- **原因**：子组件挂载时立即调用回调
- **解决**：使用 ref 跟踪之前状态，仅在变化时通知

### 6.2 响应式检测
- **工具**：Ant Design `useBreakpoint`
- **注意**：初始化时为 `undefined`，需正确处理

### 6.3 全屏覆盖
- **方式**：绝对定位 + zIndex
- **注意**：需确保覆盖在文档区域之上
