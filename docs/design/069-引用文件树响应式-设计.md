## 1. 设计目标

为 DocViewer 页面的引用文件树组件添加响应式折叠功能，在中等屏幕（lg 以下）下默认折叠，用户可手动展开/收起，提升窄屏用户的文档阅读体验。
同时在 AI 助手打开时隐藏引用文件树，减少主文档区横向挤压。

## 3. 状态管理

### 3.1 新增状态

```typescript
// 引用文件树折叠状态
const [referenceTreeCollapsed, setReferenceTreeCollapsed] = useState<boolean>(false);
```

### 3.2 折叠逻辑

```typescript
// 切换折叠状态
const toggleReferenceTree = useCallback(() => {
  setReferenceTreeCollapsed(prev => !prev);
}, []);

// 监听断点变化，当从 xl 缩小到 lg 以下时，自动折叠
useEffect(() => {
  if (!screens.xl && !referenceTreeCollapsed) {
    setReferenceTreeCollapsed(true);
  }
}, [screens.xl]);

// 初始状态：根据屏幕尺寸决定是否折叠
useEffect(() => {
  setReferenceTreeCollapsed(!screens.xl);
}, [screens.xl, docId]); // docId 变化时（如切换文档）也重新设置
```

## 4. 交互设计

### 4.1 折叠按钮

- **位置**：Card 标题栏右侧（通过 `extra` 属性）
- **图标**：
  - 折叠状态：`ChevronRight`（右箭头，表示可以展开）
  - 展开状态：`ChevronDown`（下箭头，表示可以折叠）
- **样式**：与页面其他操作按钮风格一致（`type="text"`）

### 4.2 折叠动画

- **动画时长**：250ms
- **动画类型**：最大高度过渡（max-height transition）
- **实现方式**：通过 CSS `transition` 属性控制内部容器 `max-height`

### 4.3 响应式行为

| 断点 | 默认状态 | 布局方式 |
|------|----------|----------|
| `screens.xl` (≥1200px) | 展开 | 右侧 sticky 侧边栏，宽度 320px |
| `screens.lg` (≥992px) | 折叠 | 下方 block，宽度 100% |
| `screens.md` (≥768px) | 折叠 | 下方 block，宽度 100% |
| `screens.sm` (<768px) | 折叠 | 下方 block，宽度 100% |

### 4.4 与 AI 助手联动

- 当 `copilotOpen=true` 时，引用文件树区域不渲染；
- 当 `copilotOpen=false` 时，按原有条件显示引用文件树（`!isIndexView && hasReferenceTree`）；
- 通过条件渲染避免 AI 助手与引用文件树同时占用横向空间。

## 8. 风险与注意事项

### 8.1 动画性能
- 避免使用 `height: 'auto'`，该写法无法平滑动画过渡
- 使用可动画的 `max-height` 数值或 `calc(...)` 作为替代方案

### 8.2 Tree 组件重新渲染
- 确保折叠/展开不会导致 Tree 组件重新渲染数据
- Tree 数据（`referenceTreeData`）应保持稳定

### 8.3 窗口resize事件
- Ant Design 的 `useBreakpoint` 已经处理了窗口 resize 事件
- 无需额外添加 resize 监听器
