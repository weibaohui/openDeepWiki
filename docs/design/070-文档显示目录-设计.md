## 3. 核心设计

### 3.1 标题解析

从 Markdown 原始文本中提取标题，使用正则匹配：

```
/^(#{1,3})\s+(.+)$/gm
```

提取信息：
- `level`：标题层级（1/2/3）
- `text`：标题文本
- `id`：slug（将标题文本转小写、空格替换为 `-`，去除特殊字符）

解析函数位于 `DocToc.tsx` 中，由 DocViewer 调用并将 headings 数组传入。

### 3.2 标题锚点

在 `MarkdownRender.tsx` 中，通过 `components` prop 传入自定义 `h1`/`h2`/`h3` 渲染函数，为每个标题元素添加 `id` 属性（与 slug 一致）。

```tsx
// 示例
const components = {
  h1: ({ children }) => <h1 id={slugify(children)}>{children}</h1>,
  h2: ({ children }) => <h2 id={slugify(children)}>{children}</h2>,
  h3: ({ children }) => <h3 id={slugify(children)}>{children}</h3>,
}
```

`slugify` 规则：
1. 转小写
2. 空格 → `-`
3. 去除非字母、数字、汉字、`-` 的字符
4. 连续 `-` 合并

### 3.3 滚动跳转

点击目录项时，使用 `document.getElementById(id)?.scrollIntoView({ behavior: 'smooth', block: 'start' })` 实现平滑滚动。

内容区容器为 `overflow-y: auto` 的 `div`，需确认 `scrollIntoView` 在该容器中正常工作（若不正确，改为手动计算 `scrollTop`）。

### 3.4 当前标题高亮（IntersectionObserver）

使用 `IntersectionObserver` 监听所有标题元素的可见性：

```
threshold: 0,
rootMargin: '-10% 0px -80% 0px'
```

当某标题进入视口时，将其 id 设为 `activeId`，对应目录项添加高亮样式。

### 3.5 TOC 组件 API

```tsx
interface TocHeading {
  level: number;   // 1 | 2 | 3
  text: string;
  id: string;
}

interface DocTocProps {
  headings: TocHeading[];
  activeId?: string;
}
```

组件内部通过 IntersectionObserver 管理 `activeId`，无需外部传入。

### 3.6 右侧面板布局

当前右侧面板显示引用文件树（Reference File Tree）。改造为：

```
右侧面板 (320px, sticky)
└── Ant Design Tabs
    ├── Tab "目录"   → <DocToc headings={headings} />
    └── Tab "引用文件" → 原有引用文件树内容
```

- 当 `headings` 为空时，隐藏整个右侧面板（与原逻辑 `referenceLinks.length === 0` 合并判断）
- AI 助手展开时，右侧面板已有隐藏逻辑，无需额外处理

## 4. 数据流

```
DocViewer
  ├── document.content (Markdown 原文)
  ├── parseHeadings(content) → TocHeading[]
  ├── 传入 MarkdownRender: components={{ h1, h2, h3 }} (带 id)
  └── 右侧面板
        └── Tabs
              ├── DocToc (内部管理 activeId via IntersectionObserver)
              └── 引用文件树
```

---

## 5. 样式约定

- TOC 项按层级缩进：h1 无缩进，h2 缩进 12px，h3 缩进 24px
- 激活项：主题色文字 + 左侧 2px 竖线
- 字体：12px，行高 1.6
- 最大高度：`calc(100vh - 200px)`，超出可滚动
- 整体跟随主题（浅色/深色）

---

## 6. 已知限制

- 使用 HashRouter，URL 中 `#heading-id` 与路由 hash 冲突，故不修改 URL hash，仅使用 `scrollIntoView` 实现跳转
- `XMarkdown` 的 `components` prop 支持情况需在实现阶段验证；若不支持，改为对渲染后 DOM 直接注入 id（useEffect + querySelectorAll）
- 文档切换时需重新计算标题列表和重建 IntersectionObserver
