# 任务再次生成按钮 - 实现总结

## 1. 功能概述

在仓库详情页左侧任务列表中新增“再次生成”按钮，使任务成功后也可重新触发生成流程。

## 2. 需求对应关系

| 需求点 | 实现情况 | 说明 |
| :--- | :--- | :--- |
| **新增再次生成按钮** | ✅ 已实现 | 任务成功状态显示“再次生成”操作 |
| **成功任务可再次生成** | ✅ 已实现 | 对 `succeeded/completed` 状态开放按钮 |
| **触发重试接口** | ✅ 已实现 | 复用 `POST /api/tasks/:id/retry` |
| **状态刷新与提示** | ✅ 已实现 | 触发后刷新任务列表并提示结果 |

## 3. 关键实现点

### 3.1 任务列表操作区
- 仓库详情页任务列表新增“再次生成”操作，并在成功状态展示。

### 3.2 状态与图标兼容
- 前端任务状态类型新增 `succeeded`，成功状态统一显示完成图标。

### 3.3 国际化
- 新增“再次生成”“重试”等文案的中英文配置。

## 4. 已知限制与待改进点

1. **按钮权限与限制**：当前未引入权限控制，默认所有用户可见。
2. **批量操作**：暂不支持批量再次生成。
