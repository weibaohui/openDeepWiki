## 一、背景（Why）

当前导出功能以“一个文档一个 Markdown 文件”的方式打包为压缩包，阅读体验分散，不利于线下连续阅读与分享。需要支持将仓库下的全部文档内容合并导出为单个 PDF，提升可读性与交付效率。

## 二、目标（What，必须可验证）

- [ ] 提供“导出 PDF”能力，将同一仓库的全部文档合并为一个 PDF 文件
- [ ] PDF 内容顺序与现有文档目录/列表顺序一致
- [ ] 支持浏览器直接下载生成的 PDF 文件
- [ ] 导出过程中出现错误时，返回可识别的错误信息

## 三、非目标（Explicitly Out of Scope）

- 不要求支持单个文档独立导出 PDF
- 不要求支持自定义 PDF 模板或主题
- 不要求支持导出 PPT/闪卡等其他格式
- 不新增或修改现有文档内容生成逻辑

## 四、详细需求

### 4.1 后端能力

- 新增导出 PDF 接口，输入为 repository_id
- 服务端负责聚合该仓库下所有文档内容，并生成单个 PDF
- 返回内容类型为 application/pdf，支持浏览器下载

### 4.2 前端交互

- 在查看文档页新增“导出 PDF”按钮，下载按钮拓展为下拉两个选项，一个是zip方式，一个是PDF方式
- 点击后发起导出请求并触发浏览器下载
- 下载文件名包含仓库名称与时间戳或固定后缀

### 4.3 内容组织

- PDF 中按文档列表顺序拼接文档内容
- 每个文档应有清晰分隔（标题或分页）
- 若文档内容为空，仍保留标题占位

## 五、验收标准

1. 点击“导出 PDF”后可下载单个 PDF 文件
2. PDF 内包含该仓库全部文档内容且顺序正确
3. 导出失败时前端提示明确错误
4. 不影响现有 Markdown 压缩包导出功能
