## 1. 实现概述
在增量分析任务完成后发布“增量更新完成”事件，订阅端接收事件并更新仓库最新分支与提交信息，确保后续增量分析以最新 commit 为基线。

## 2. 需求对应关系

### 2.1 需求来源

| :--- | :--- | :--- |
| git pull 后基于基线提交进行增量分析 | 已完成 | 增量任务执行完成后发布事件，更新基线 commit |
| 后端输出中文日志 | 已完成 | 增量更新完成事件发布与提交更新均记录中文日志 |
| 接口返回成功或失败 | 已完成 | 事件发布失败将返回错误并记录日志 |

## 3. 关键实现点

### 3.1 增量更新完成事件发布
- 在增量分析任务完成后读取最新分支与提交信息。
- 发布仓库增量更新完成事件，携带 branch 与 commit。

### 3.2 事件订阅与 commit 落库
- 新增增量更新完成事件类型并注册订阅处理器。
- 订阅端调用仓库服务更新 `CloneBranch` 与 `CloneCommit`。

### 3.3 仓库提交更新服务方法
- 新增仓库提交信息更新方法，包含提交空值校验与日志输出。

## 4. 测试记录
- 后端测试：`go test ./...`

## 5. 安全自检
1. **权限风险**：未新增外部访问入口，仅在内部事件处理链路中更新仓库信息。
2. **输入校验**：对 repoID 与 commit 空值进行校验，异常直接返回错误。
3. **敏感信息**：日志仅输出分支与提交哈希，不包含密钥或凭证。
