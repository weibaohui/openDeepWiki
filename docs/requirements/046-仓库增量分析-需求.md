## 背景与目标

为仓库详情页面提供“增量分析”入口，支持基于数据库记录的仓库 commit 基线，在本地仓库执行 git pull 后进行增量变更分析，并将变更内容在后台打印，便于运维与调试。

## 需求范围

### 功能需求

1. 在仓库详情页“更多操作”菜单抽屉中新增“仓库增量分析”按钮。
2. 点击按钮后，前端调用后端接口，传入 repoID。
3. 后端根据 repoID 查询数据库，获取：
   - 仓库本地路径
   - 基线 commit（base commit）
4. 后端在本地仓库执行 git pull，确保拉取成功并覆盖本地仓库。
5. 后端基于 base commit 执行增量变更分析，获得变更内容。
6. 后端以中文日志在后台打印增量变更内容，便于排查。

### 非功能需求

1. 接口需返回明确的成功或失败响应。
2. 若仓库路径不存在或 git pull 失败，需返回错误并记录日志。
3. 变更内容仅在后端日志输出，不强制返回给前端。

## 不在范围内

1. 增量变更内容展示在前端页面。
2. 对增量变更内容进行 AI 分析或二次处理。
3. 多仓库批量增量分析。

## 验收标准

1. 菜单抽屉中出现“仓库增量分析”按钮。
2. 点击后触发后端接口，接口返回成功。
3. 后端能够完成 git pull 并进行增量分析。
4. 后端日志可看到增量变更内容（中文描述）。
