## 背景与目标

当前系统已实现仓库增量分析功能（046-仓库增量分析），并在数据库中记录了增量更新历史（IncrementalUpdateHistory 表）。然而，用户无法在前端界面查看这些历史记录，无法了解仓库的增量更新情况和变更统计。

本需求旨在为仓库详情页面提供增量同步历史查看功能，让用户能够清晰地了解每次增量更新的时间、变更范围和统计信息。

## 需求范围

### 功能需求

1. **入口设计**
   - 在仓库详情页"更多操作"菜单抽屉中新增"增量同步历史"按钮
   - 点击按钮后打开增量同步历史查看界面

2. **历史记录列表展示**
   - 以列表形式展示该仓库的所有增量同步历史记录
   - 每条记录包含以下信息：
     - 更新时间（created_at）
     - 基线提交ID（base_commit）
     - 最新提交ID（latest_commit）
     - 新增目录数（added_dirs）
     - 更新目录数（updated_dirs）

3. **界面设计要求**
   - 保持与现有页面风格一致（使用 Ant Design 组件）
   - 界面美观大方，响应式布局
   - 空状态时显示友好提示
   - 支持移动端访问

4. **数据获取**
   - 前端调用后端 API 获取指定仓库的增量同步历史记录
   - 后端按时间倒序返回记录（最新的在前）

### 非功能需求

1. **性能要求**
   - API 响应时间应在 1 秒以内
   - 支持分页或限制返回数量，避免一次性返回过多数据

2. **用户体验**
   - 加载时显示 loading 状态
   - 错误时显示友好的错误提示
   - 列表项应易于阅读，重要信息突出显示

3. **国际化**
   - 所有界面文本支持国际化（中英文）

## 不在范围内

1. 增量同步历史的删除功能
2. 增量同步历史的详情查看（如查看具体变更的文件列表）
3. 增量同步历史的统计分析图表
4. 增量同步历史的导出功能

## 验收标准

1. 仓库详情页"更多操作"菜单中出现"增量同步历史"按钮
2. 点击按钮能够打开增量同步历史查看界面
3. 界面能够正确显示该仓库的所有增量同步历史记录
4. 每条记录显示完整的信息（时间、commit ID、目录统计）
5. 界面风格与现有页面保持一致
6. 支持中英文切换
7. 移动端可正常访问和使用
