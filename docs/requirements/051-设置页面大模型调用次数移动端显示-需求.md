## 1. 背景（Why）

当前系统在设置页面（API Key Management）中显示每个大模型的调用次数（request_count），但存在以下问题：

1. **响应式显示不一致**：PC 端（屏幕宽度 >= 992px）可以看到 `Request Count` 列，但手机端看不到该列
2. **用户体验问题**：移动端用户无法查看各个模型的调用次数，影响使用体验

该问题是由于 Ant Design Table 组件的 `responsive: ['lg']` 属性限制了该列只在大屏幕上显示。

---

## 2. 目标（What，必须可验证）

- [ ] 移除 `request_count` 列的响应式显示限制
- [ ] PC 端和移动端都能看到每个大模型的调用次数
- [ ] 保持现有的表格布局在 PC 端的显示效果

---

## 3. 非目标（Explicitly Out of Scope）

- [ ] 不修改其他列的显示逻辑
- [ ] 不改变表格的滚动行为
- [ ] 不修改后端接口和数据结构

---

## 4. 核心概念定义

### 4.1 响应式断点（Ant Design Grid）

Ant Design 使用以下断点进行响应式设计：

| 断点 | 屏幕宽度范围 | 适用设备 |
|------|-------------|---------|
| xs | < 576px | 手机 |
| sm | >= 576px | 手机（横屏） |
| md | >= 768px | 平板 |
| lg | >= 992px | PC/笔记本 |
| xl | >= 1200px | 大屏 PC |
| xxl | >= 1600px | 超大屏 |

当前问题：`request_count` 列配置了 `responsive: ['lg']`，导致只在 lg 及以上屏幕显示。

```typescript

    title: t('apiKey.request_count', 'Request Count'),
    dataIndex: 'request_count',
    key: 'request_count',
    render: (text: number) => text ? text.toString() : '-',
    responsive: ['lg']
```
