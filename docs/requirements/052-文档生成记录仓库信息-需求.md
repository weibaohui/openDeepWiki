# 1. 背景（Why）

当前系统生成文档时，仅记录了文档的基本信息（如标题、内容、版本等），但没有记录生成文档时所使用的代码仓库的分支和对应的 commit id。这导致用户无法追溯文档是基于哪个版本的代码生成的，降低了文档的可追溯性和可信度。

为了解决这个问题，需要：
1. 在生成文档时，记录当前仓库的分支和 commit id
2. 在文档查看页面底部显示这些仓库信息，类似 TokenUsage 的显示方式

# 2. 目标（What，必须可验证）

- [ ] 在 Document 模型中添加 clone_branch 和 clone_commit_id 字段
- [ ] 在文档生成时，将仓库当前的 clone_branch 和 clone_commit_id 保存到 Document 表
- [ ] 在文档查看页面（/repo/:id/doc/:docId）底部显示仓库信息
- [ ] 仓库信息显示块样式参考 TokenUsage 的显示块
- [ ] 显示内容包括：分支名称、commit id（完整显示）

# 3. 非目标（Explicitly Out of Scope）

- 不修改现有文档版本管理功能
- 不提供仓库信息的编辑功能
- 不实现分支切换功能
- 不提供 commit 历史对比功能
- 不在文档概览页面显示这些信息

# 4. 使用场景 / 用户路径

## 场景 1：文档生成时记录仓库信息

```
用户在仓库详情页点击"执行所有任务"
                ↓
系统开始生成文档
                ↓
文档生成完成，保存到数据库
                ↓
系统将当前仓库的 clone_branch 和 clone_commit_id 保存到 Document 表
                ↓
文档显示在文档列表中
```

## 场景 2：用户查看文档时看到仓库信息

```
用户访问文档查看页面 /repo/5/doc/10
                ↓
页面显示文档内容
                ↓
页面底部显示 TokenUsage 信息块
                ↓
页面底部同时显示仓库信息块
                ↓
用户可以看到生成文档时的分支和 commit id
```

# 9. 验收标准（Acceptance Criteria）

## 9.1 功能验收
- [ ] 如果文档生成时仓库有 clone_branch 和 clone_commit_id，则这些信息被保存到 Document 表
- [ ] 如果文档生成时仓库没有 clone_branch 和 clone_commit_id，则这些字段为空
- [ ] 如果用户访问文档页面，则页面底部显示仓库信息（如果存在）
- [ ] 如果文档没有仓库信息，则不显示仓库信息块

## 9.2 数据验收
- [ ] 如果保存文档成功，则 Document 表中包含 clone_branch 和 clone_commit_id 字段
- [ ] Document 的 clone_branch 值等于关联 Repository 的 clone_branch 值
- [ ] Document 的 clone_commit_id 值等于关联 Repository 的 clone_commit_id 值

## 9.3 UI 验收
- [ ] 仓库信息显示块位于 TokenUsage 显示块下方
- [ ] 仓库信息显示块样式与 TokenUsage 显示块一致
- [ ] 分支名称正确显示
- [ ] commit id 完整显示（不截断）
- [ ] 使用合适的图标区分分支和 commit
- [ ] 支持中英文切换

## 9.4 API 验收
- [ ] GET /api/documents/:id 返回的 Document 对象包含 clone_branch 和 clone_commit_id
- [ ] GET /api/repositories/:id/documents 返回的 Document 列表包含 clone_branch 和 clone_commit_id

# 10. 风险与已知不确定点

## 已知风险
1. **历史数据兼容性**
   - 说明：已存在的 Document 记录没有 clone_branch 和 clone_commit_id
   - 影响：这些记录的仓库信息为空
   - 应对：数据库迁移时，尝试从关联 Repository 填充默认值

2. **Repository 信息变更**
   - 说明：Repository 的 clone_branch 和 clone_commit_id 可能被更新
   - 影响：已生成的文档的仓库信息不会同步更新
   - 应对：这是预期行为，文档生成时的快照应保持不变

## 不确定点
无
