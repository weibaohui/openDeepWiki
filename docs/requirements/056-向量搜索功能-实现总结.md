## 二、功能实现总结

### 2.1 已实现功能

#### 2.1.1 数据模型层

**文件**：`backend/internal/model/models.go`

**新增模型**：
- `DocumentVector` - 文档向量模型，存储向量嵌入数据
- `VectorTask` - 向量生成任务模型，跟踪异步任务状态

**数据库迁移**：
- `backend/internal/pkg/database/database.go` - 添加了新表的自动迁移

#### 2.1.2 向量仓储层

**文件**：`backend/internal/repository/vector_repo.go`

**实现的接口**：
- `VectorRepository` - 向量数据的增删改查
- `VectorTaskRepository` - 向量任务的增删改查

**核心方法**：
- `Create()` - 创建向量记录
- `GetByDocumentID()` - 获取文档向量
- `GetByDocumentIDAndModel()` - 获取指定模型的向量
- `Delete()` / `DeleteByDocumentID()` - 删除向量
- `GetAll()` - 获取所有向量
- `GetVectorizedCount()` - 获取已向量化文档数
- `GetStatus()` - 获取向量生成状态统计
- `BatchCreate()` - 批量创建向量

#### 2.1.3 领域层（向量接口）

**文件**：`backend/internal/domain/vector/embedding.go`

**定义的接口**：
- `EmbeddingProvider` - 向量嵌入提供者接口

**核心方法**：
- `Embed()` - 生成单个文本的向量
- `EmbedBatch()` - 批量生成向量
- `Dimension()` - 返回向量维度
- `ModelName()` - 返回模型名称
- `HealthCheck()` - 检查提供者可用性

#### 2.1.4 向量索引层

**文件**：`backend/internal/pkg/vector/index.go`

**定义的接口**：
- `VectorIndex` - 向量索引接口

**核心实现**：
- `FlatIndex` - 暴力搜索索引实现（用于小规模数据）
- `Search()` - 搜索最近的 k 个向量
- `Add()` / `AddBatch()` - 添加向量
- `Remove()` - 删除向量
- `Size()` - 返回索引大小
- `cosineDistance()` - 余弦相似度计算
- `euclideanDistance()` - 欧氏距离计算

**注**：当前使用 FlatIndex 实现，后续可扩展为 HNSW 获得更好性能。

#### 2.1.5 OpenAI 嵌入提供者

**文件**：`backend/internal/service/vector/embedding/openai.go`

**核心方法**：
- `NewOpenAIEmbeddingProvider()` - 创建提供者实例
- `Embed()` - 调用 OpenAI Embeddings API
- `EmbedBatch()` - 批量调用 API
- `Dimension()` - 返回向量维度（1536 或 3072）
- `ModelName()` - 返回模型名称
- `HealthCheck()` - 检查 API 可用性

**特性**：
- 支持自定义 BaseURL 和 API Key
- 支持可配置超时时间
- 详细的日志输出（使用 `klog.V(6)`）

#### 2.1.6 向量搜索服务

**文件**：`backend/internal/service/vector/search_service.go`

**核心方法**：
- `Search()` - 语义搜索，支持过滤条件
- `FindSimilarDocuments()` - 查找相似文档
- `RebuildIndex()` - 重建向量索引
- `applyFilters()` - 应用过滤条件（repository_id、is_latest 等）

**特性**：
- 余弦相似度计算
- 内容片段生成
- 过滤条件支持

#### 2.1.7 向量生成服务

**文件**：`backend/internal/service/vector/embedding_service.go`

**核心方法**：
- `GenerateForDocument()` - 为文档生成向量（创建任务）
- `GenerateForRepository()` - 批量为仓库生成向量
- `RegenerateForDocument()` - 重新生成文档的向量
- `GetStatus()` - 获取向量生成状态
- `RebuildIndex()` - 重建索引
- `Start()` - 启动后台任务处理协程
- `worker()` - 单个任务处理协程

**特性**：
- 后台异步处理（goroutine + channel）
- 错误处理和重试
- 任务状态跟踪（pending/processing/completed/failed）

#### 2.1.8 向量事件订阅器

**文件**：`backend/internal/eventbus/vector_subscriber.go`

**核心方法**：
- `Subscribe()` - 订阅文档保存和更新事件
- `handleDocSaved()` - 处理文档保存事件
- `handleDocUpdated()` - 处理文档更新事件

**特性**：
- 自动触发向量生成
- 异步处理不阻塞主流程
- 详细的日志记录

### 2.2 API 接口层

**文件**：`backend/internal/handler/vector.go`

**实现的接口**：
| 方法 | 路径 | 说明 |
|------|------|------|
| `Search` | POST /api/vectors/search | 语义搜索 |
| `FindSimilarDocuments` | GET /api/documents/:id/similar | 查找相似文档 |
| `GenerateVector` | POST /api/documents/:id/vector/generate | 手动生成向量 |
| `GenerateRepositoryVectors` | POST /api/repositories/:id/vectors/generate | 批量生成向量 |
| `RegenerateVector` | POST /api/documents/:id/vector/regenerate | 重新生成向量 |
| `GetVectorStatus` | GET /api/vectors/status | 获取状态 |
| `DeleteVector` | DELETE /api/documents/:id/vector | 删除向量 |

**请求/响应结构**：
- `SearchRequest` - 搜索请求（query, model, repository_id, top_k, min_similarity, filters）
- `SearchResultDTO` - 搜索结果（document_id, title, repository_id, similarity, snippet）
- `VectorStatusDTO` - 向量状态（total_documents, vectorized_count, pending_count, failed_count, processing_count）

### 2.3 事件集成

**修改文件**：
- `backend/internal/eventbus/doc_event.go` - 添加文档保存/更新事件类型
- `backend/internal/service/document.go` - 发布文档事件

**新增事件类型**：
- `DocEventSaved` - 文档保存事件
- `DocEventUpdated` - 文档更新事件

**文档事件结构扩展**：
- 添加 `Title` 字段
- 添加 `Content` 字段

**集成方式**：
- 通过事件总线解耦向量生成和文档保存
- 异步任务处理不影响主流程

**预期配置**（在 `config.yaml` 中添加）：

  index_type: "hnsw"

## 三、技术决策

### 3.1 向量存储方案

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 向量存储 | SQLite BLOB | 利用现有数据库，无额外依赖 |
| 索引实现 | FlatIndex（暴力搜索） | 当前使用简单实现，可扩展为 HNSW |
| 未来扩展 | HNSW 库 | 后续可通过 `github.com/creachadair/go-hnsw` 获得更好性能 |

### 3.2 向量生成器

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 首选实现 | OpenAI Embeddings API | 成熟稳定，生态完善 |
| 接口抽象 | EmbeddingProvider | 支持多种后端扩展 |
| 扩展能力 | Ollama、通用 HTTP API | P1/P2 优先级 |

### 3.3 异步任务处理

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 队列实现 | goroutine + channel | Go 原生并发模型 |
| 错误隔离 | 向量生成失败不影响文档保存 | 保证核心流程稳定 |
| 任务跟踪 | VectorTask 表 | 独立状态管理 |

## 六、使用说明

### 6.1 启动向量生成服务

在主程序启动后，向量生成服务会自动启动后台协程。

### 6.2 语义搜索

调用 POST `/api/vectors/search` 接口：

```json
{
  "query": "容器化技术",
  "top_k": 10,
  "min_similarity": 0.7
  "repository_id": 123
}
```

### 6.3 相似文章推荐

调用 GET `/api/documents/{docID}/similar` 接口：

```bash
curl http://localhost:8080/api/documents/123/similar?top_k=5&min_similarity=0.7
```

### 6.4 手动生成向量

调用 POST `/api/documents/{docID}/vector/generate` 接口：

```bash
curl -X POST http://localhost:8080/api/documents/123/vector/generate
```

### 6.5 批量生成向量

调用 POST `/api/repositories/{repoID}/vectors/generate` 接口：

```bash
curl -X POST http://localhost:8080/api/repositories/123/vectors/generate
```

### 6.6 查看向量状态

调用 GET `/api/vectors/status` 接口：

```bash
curl http://localhost:8080/api/vectors/status
```
