## 二、功能实现总结

### 2.1 已实现功能

#### 2.1.1 数据模型层

**文件**：`backend/internal/model/models.go`

**新增模型**：
- `DocumentVector` - 文档向量模型，存储向量元数据
- `VectorTask` - 向量生成任务模型，跟踪异步任务状态

**数据库迁移**：
- `backend/internal/pkg/database/database.go` - 添加了新表的自动迁移

#### 2.1.2 sqvect 向量存储封装

**文件**：
- `backend/internal/pkg/sqvect/store.go` - 存储封装
- `backend/internal/pkg/sqvect/manager.go` - 管理器

**核心功能**：
- `NewStore()` - 创建向量存储实例
- `Upsert()` - 插入或更新向量
- `UpsertBatch()` - 批量插入向量
- `Search()` - 向量相似度搜索
- `HybridSearch()` - 混合搜索（向量 + 关键词）
- `GetByID()` / `GetByDocID()` - 获取向量
- `Delete()` / `DeleteByDocID()` - 删除向量
- `Stats()` - 获取存储统计信息

**配置选项**：
- `Path` - 数据库文件路径
- `Dimensions` - 向量维度（默认 2560）
- `IndexType` - 索引类型（HNSW 或 IVF）
- `CollectionName` - 集合名称

#### 2.1.3 向量仓储层（双数据库架构）

**文件**：
- `backend/internal/repository/vector_repo.go` - GORM 元数据仓储
- `backend/internal/repository/sqvect_vector_repo.go` - sqvect 向量仓储

**实现的接口**：
- `VectorRepository` - 向量数据的增删改查
- `VectorTaskRepository` - 向量任务的增删改查

**核心方法**：
- `Create()` - 创建向量记录（同时写入 sqvect 和 GORM）
- `GetByDocumentID()` - 获取文档向量
- `GetByDocumentIDAndModel()` - 获取指定模型的向量
- `Delete()` / `DeleteByDocumentID()` - 删除向量
- `GetAll()` - 获取所有向量
- `GetVectorizedCount()` - 获取已向量化文档数
- `GetStatus()` - 获取向量生成状态统计
- `BatchCreate()` - 批量创建向量

#### 2.1.4 领域层（向量接口）

**文件**：`backend/internal/domain/vector/embedding.go`

**定义的接口**：
- `EmbeddingProvider` - 向量嵌入提供者接口

**核心方法**：
- `Embed()` - 生成单个文本的向量
- `EmbedBatch()` - 批量生成向量
- `Dimension()` - 返回向量维度
- `ModelName()` - 返回模型名称
- `HealthCheck()` - 检查提供者可用性

#### 2.1.5 通用嵌入提供者

**文件**：`backend/internal/service/vector/embedding/openai.go`

**核心方法**：
- `NewOpenAIEmbeddingProvider()` - 创建提供者实例（支持 OpenAI 兼容 API）
- `Embed()` - 调用嵌入 API
- `EmbedBatch()` - 批量调用 API
- `Dimension()` - 返回向量维度（默认 2560，支持配置）
- `ModelName()` - 返回模型名称
- `HealthCheck()` - 检查 API 可用性

**特性**：
- 支持 OpenAI 兼容的 API（包括 Qwen3-Embedding-4B）
- 支持自定义 BaseURL 和 API Key
- 支持可配置超时时间
- 从数据库动态加载配置
- 详细的日志输出（使用 `klog.V(6)`）

#### 2.1.6 向量搜索服务

**文件**：`backend/internal/service/vector/search_service.go`

**核心方法**：
- `Search()` - 语义搜索，支持过滤条件
- `FindSimilarDocuments()` - 查找相似文档
- `searchWithSqvect()` - 使用 sqvect 进行高效向量搜索
- `searchInMemory()` - 内存搜索（回退方案）

**特性**：
- 优先使用 sqvect 进行高效向量搜索
- 支持内存搜索作为回退方案
- 余弦相似度计算
- 内容片段生成
- 过滤条件支持

#### 2.1.7 向量生成服务

**文件**：`backend/internal/service/vector/embedding_service.go`

**核心方法**：
- `GenerateForDocument()` - 为文档生成向量（创建任务）
- `GenerateForRepository()` - 批量为仓库生成向量
- `RegenerateForDocument()` - 重新生成文档的向量
- `GetStatus()` - 获取向量生成状态
- `Start()` - 启动后台任务处理协程
- `worker()` - 单个任务处理协程

**特性**：
- 后台异步处理（goroutine + channel）
- 错误处理和重试
- 任务状态跟踪（pending/processing/completed/failed）

#### 2.1.8 向量事件订阅器

**文件**：`backend/internal/eventbus/vector_subscriber.go`

**核心方法**：
- `Subscribe()` - 订阅文档保存和更新事件
- `handleDocSaved()` - 处理文档保存事件
- `handleDocUpdated()` - 处理文档更新事件

**特性**：
- 自动触发向量生成
- 异步处理不阻塞主流程
- 详细的日志记录

### 2.2 API 接口层

**文件**：`backend/internal/handler/vector.go`

**实现的接口**：

|------|------|------|
| `Search` | POST /api/vectors/search | 语义搜索 |

## 三、技术决策

### 3.1 向量存储方案

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 向量数据库 | sqvect | 纯 Go 实现，无需 CGO，支持 HNSW/IVF 索引 |
| 数据持久化 | SQLite 文件 | 单一文件，易于备份和迁移 |
| 双数据库架构 | GORM + sqvect | 业务数据与向量数据分离，职责清晰 |

### 3.2 向量生成器

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 首选实现 | Qwen3-Embedding-4B | 阿里云通义千问嵌入模型，40 亿参数，32K 上下文，2560 维向量 |
| 接口抽象 | EmbeddingProvider | 支持多种后端扩展 |
| 扩展能力 | 通用 HTTP API、Ollama | P1/P2 优先级 |

### 3.3 异步任务处理

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 队列实现 | goroutine + channel | Go 原生并发模型 |
| 错误隔离 | 向量生成失败不影响文档保存 | 保证核心流程稳定 |
| 任务跟踪 | VectorTask 表 | 独立状态管理 |

## 六、使用说明

### 6.1 初始化 sqvect 存储

```go
import "github.com/weibaohui/opendeepwiki/backend/internal/pkg/sqvect"

// 创建配置
config := sqvect.DefaultConfig("data/vectors.db")
config.Dimensions = 2560  // Qwen3-Embedding-4B 维度

// 创建存储
store, err := sqvect.NewStore(config)
if err != nil {
    panic(err)
}
defer store.Close()
```

### 6.2 启动向量生成服务

在主程序启动后，向量生成服务会自动启动后台协程。

### 6.3 语义搜索

调用 POST `/api/vectors/search` 接口：

```json
{
  "query": "容器化技术",
  "top_k": 10,
  "min_similarity": 0.7,
  "repository_id": 123
}
```

### 6.4 相似文章推荐

调用 GET `/api/documents/{docID}/similar` 接口：

```bash
curl http://localhost:8080/api/documents/123/similar?top_k=5&min_similarity=0.7
```

### 6.5 手动生成向量

调用 POST `/api/documents/{docID}/vector/generate` 接口：

```bash
curl -X POST http://localhost:8080/api/documents/123/vector/generate
```

### 6.6 批量生成向量

调用 POST `/api/repositories/{repoID}/vectors/generate` 接口：

```bash
curl -X POST http://localhost:8080/api/repositories/123/vectors/generate
```

### 6.7 查看向量状态

调用 GET `/api/vectors/status` 接口：

```bash
curl http://localhost:8080/api/vectors/status
```
