# 文章向量搜索功能需求文档

## 1. 需求概述

为 openDeepWiki 项目引入向量搜索能力，为每篇文章生成向量嵌入，实现基于语义的相似文章检索功能。支持多种向量模型后端，使用 sqvect 作为向量数据库，实现双数据库架构。

## 2. 技术选型

### 2.1 向量存储方案

| 方案 | 状态 | 理由 |
|------|------|------|
| **sqvect** | 已选 | 纯 Go 实现的轻量级向量数据库，基于 SQLite，无需 CGO，支持 HNSW/IVF 索引 |

**sqvect 特点：**
- 100% 纯 Go 实现，无需 CGO，易于跨平台编译
- 基于 SQLite，数据持久化存储在单一文件中
- 支持 HNSW（高精度）和 IVF（高性能）索引
- 内置 SQ8 量化，减少 75% 内存占用
- 支持混合搜索（向量 + 全文搜索 FTS5）
- 支持多集合（多租户隔离）

### 2.2 双数据库架构

| 数据库 | 用途 | 存储内容 |
|--------|------|----------|
| **GORM (SQLite/MySQL)** | 传统数据 | 用户、仓库、文档、任务、配置等业务数据 |
| **sqvect** | 向量数据 | 文档向量嵌入、向量索引 |

### 2.3 向量模型后端

支持多种模型后端，通过接口抽象：

| 后端 | 实现优先级 | 说明 |
|------|-----------|------|
| Qwen3-Embedding-4B | P0 | 阿里云通义千问嵌入模型，40 亿参数，支持 32K 上下文，2560 维向量 |
| 自定义 HTTP API | P1 | 支持任意兼容 OpenAI 格式的 API（包括其他嵌入模型） |
| Ollama 本地模型 | P2 | 支持本地部署的嵌入模型 |

**Qwen3-Embedding-4B 模型特点：**
- 基于 Qwen3 系列的密集基础模型
- 40 亿参数规模
- 支持长达 32K 的上下文长度
- 生成 2560 维的嵌入向量
- 专为文本嵌入和排序任务设计

## 3. 数据模型设计

### 3.1 GORM 数据库表（元数据）

```sql
-- 文档向量元数据表（用于关联查询和状态追踪）
CREATE TABLE document_vectors (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    document_id INTEGER NOT NULL,           -- 关联 documents.id
    model_name TEXT NOT NULL,               -- 使用的模型名称
    dimension INTEGER NOT NULL,             -- 向量维度
    generated_at DATETIME NOT NULL,         -- 生成时间
    metadata TEXT,                          -- 额外元数据（JSON）
    FOREIGN KEY (document_id) REFERENCES documents(id) ON DELETE CASCADE
);

CREATE INDEX idx_doc_vectors_document_id ON document_vectors(document_id);
CREATE INDEX idx_doc_vectors_model ON document_vectors(model_name);

-- 向量生成任务表
CREATE TABLE vector_tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    document_id INTEGER NOT NULL,
    status TEXT NOT NULL DEFAULT 'pending', -- pending/processing/completed/failed
    error_message TEXT,
    created_at DATETIME NOT NULL,
    started_at DATETIME,
    completed_at DATETIME,
    FOREIGN KEY (document_id) REFERENCES documents(id) ON DELETE CASCADE
);

CREATE INDEX idx_vector_tasks_status ON vector_tasks(status);
```

### 3.2 sqvect 向量存储

sqvect 以独立的 SQLite 文件存储向量数据，包含：
- 向量嵌入（DocumentEmbedding）
- HNSW/IVF 索引
- 全文搜索索引（FTS5）

### 3.3 数据模型 (Go)

```go
// DocumentVector 文档向量模型（GORM 元数据）
type DocumentVector struct {
    ID          uint      `json:"id" gorm:"primaryKey"`
    DocumentID  uint      `json:"document_id" gorm:"index;not null"`
    ModelName   string    `json:"model_name" gorm:"size:100;not null"`
    Vector      []float32 `json:"-" gorm:"type:blob;not null"` // sqvect 中存储
    Dimension   int       `json:"dimension" gorm:"not null"`
    GeneratedAt time.Time `json:"generated_at" gorm:"not null"`
    Metadata    string    `json:"metadata" gorm:"type:text"` // JSON 字符串
}

// VectorTask 向量生成任务
type VectorTask struct {
    ID           uint       `json:"id" gorm:"primaryKey"`
    DocumentID   uint       `json:"document_id" gorm:"index;not null"`
    Status       string     `json:"status" gorm:"size:20;not null;default:'pending'"`
    ErrorMessage string     `json:"error_message" gorm:"type:text"`
    CreatedAt    time.Time  `json:"created_at" gorm:"not null"`
    StartedAt    *time.Time `json:"started_at"`
    CompletedAt  *time.Time `json:"completed_at"`
}

// sqvect 封装（backend/internal/pkg/sqvect/store.go）
type DocumentEmbedding struct {
    ID       string            `json:"id"`       // 文档 ID 字符串
    Vector   []float32         `json:"vector"`   // 向量数据
    Content  string            `json:"content"`  // 文本内容
    DocID    string            `json:"docId"`    // 文档 ID
    Metadata map[string]string `json:"metadata"` // 元数据
    ACL      []string          `json:"acl"`      // 访问控制列表
}
```

## 4. 核心功能

### 4.1 向量生成

#### 4.1.1 自动生成（异步）

- **触发时机**：新文档保存后（`is_latest = true` 的 Document）
- **实现方式**：后台任务队列，异步处理不阻塞主流程
- **向量化内容**：`Title + "\n" + Content`

#### 4.1.2 手动触发

- **API 接口**：
  ```
  POST /api/documents/:id/vector/generate
  POST /api/repositories/:id/vectors/generate  // 批量生成
  ```
- **CLI 命令**：
  ```bash
  ./backend vector generate --doc-id <id>
  ./backend vector generate --repo-id <id> --all
  ```

#### 4.1.3 批量处理已有文档

- 支持为历史文档批量生成向量
- 支持指定模型重新生成

### 4.2 向量搜索

#### 4.2.1 语义搜索 API

```
POST /api/vectors/search
Request Body:
{
  "query": "搜索关键词或句子",
  "model": "text-embedding-v3",  // 可选，默认配置
  "repository_id": 123,           // 可选，限定仓库范围
  "top_k": 10,                    // 返回结果数，默认 10
  "min_similarity": 0.7,          // 相似度阈值，默认 0.5
  "filters": {                    // 可选过滤条件
    "is_latest": true
  }
}

Response:
{
  "results": [
    {
      "document_id": 1,
      "title": "文档标题",
      "repository_id": 123,
      "repository_name": "仓库名",
      "similarity": 0.95,
      "snippet": "内容片段..."
    },
    ...
  ]
}
```

#### 4.2.2 相似文章推荐

```
GET /api/documents/:id/similar?top_k=5&min_similarity=0.7
```

### 4.3 向量管理

#### 4.3.1 向量状态查询

```
GET /api/vectors/status
Response:
{
  "total_documents": 1000,
  "vectorized_count": 850,
  "pending_count": 50,
  "failed_count": 10,
  "processing_count": 90
}
```

#### 4.3.2 向量删除

```
DELETE /api/documents/:id/vector
```

### 5.3 向量搜索流程

1. 用户发起搜索请求
2. 使用嵌入提供者生成查询向量
3. 调用 sqvect 的 Search 或 HybridSearch 方法
4. sqvect 使用 HNSW/IVF 索引进行高效向量搜索
5. 返回相似文档 ID 和相似度分数
6. 从 GORM 数据库获取文档详情
7. 组装返回结果

### 5.4 嵌入生成器接口

```go
// EmbeddingProvider 向量嵌入提供者接口
type EmbeddingProvider interface {
    // 生成单个文本的嵌入
    Embed(ctx context.Context, text string) ([]float32, error)

    // 批量生成嵌入
    EmbedBatch(ctx context.Context, texts []string) ([][]float32, error)

    // 获取向量维度
    Dimension() int

    // 获取模型名称
    ModelName() string

    // 检查可用性
    HealthCheck(ctx context.Context) error
}
```

### 5.5 配置管理

嵌入模型配置通过数据库管理（`embedding_keys` 表），支持：
- 多配置管理：可添加多个嵌入模型配置
- 优先级控制：按优先级自动选择可用配置
- 状态管理：支持启用/禁用配置
- 使用统计：记录请求次数、错误次数、最后使用时间

**数据库配置字段**：
| 字段 | 说明 |
|------|------|
| name | 配置名称（唯一） |
| provider | 提供者类型（openai, ollama, http） |
| base_url | API 地址 |
| api_key | API Key |
| model | 模型名称 |
| dimension | 向量维度（Qwen3-Embedding-4B 默认 2560） |
| priority | 优先级（数字越小优先级越高） |
| status | 状态（enabled/disabled） |
| timeout | 超时时间（秒） |

## 7. 待确认问题

1. **向量维度**：
   - Qwen3-Embedding-4B: 2560 维（默认）
   - 支持通过配置自定义其他嵌入模型的维度
   - 不支持混合维度（同一系统内应使用相同维度的模型）

2. **索引更新策略**：
   - 文档更新时如何处理旧向量？（替换、保留版本、删除？）

3. **前端集成范围**：
   - 是否需要在文档详情页显示相似文章？
   - 是否需要独立的搜索页面？

## 8. 非功能需求

- **性能**：100 万文档下，搜索响应时间 < 100ms
- **可靠性**：向量生成失败不影响文档保存
- **兼容性**：保持向后兼容，不影响现有功能
- **可扩展性**：支持新增向量模型提供者
