## 2. 后端实现

### 2.1 数据模型 (`backend/internal/model/models.go`)

新增三个数据表：
- `chat_sessions`: 会话表，存储会话基本信息
- `chat_messages`: 消息表，存储对话消息
- `chat_tool_calls`: 工具调用表，存储工具调用详情

### 2.2 Repository 层

- `chat_session_repo.go`: 会话的 CRUD 操作
- `chat_message_repo.go`: 消息的 CRUD 操作，支持流式更新
- `chat_tool_call_repo.go`: 工具调用的 CRUD 操作

### 2.3 Service 层

- `chat_service.go`: 对话业务逻辑，包括：
  - 会话创建、查询、删除
  - 消息创建、更新、完成
  - 工具调用记录管理

### 2.4 Handler 层

- `chat_handler.go`: REST API 和 WebSocket 处理器
  - 会话管理 API (POST/GET/DELETE)
  - WebSocket 连接管理
  - 消息收发处理
  - 停止生成控制
- `chat_hub.go`: WebSocket Hub，管理连接生命周期
- `chat_agent_executor.go`: Agent 执行器框架

### 2.5 Agent 定义

- `backend/agents/chat_assistant.yaml`: 问答 Agent 配置

### 2.6 路由注册

- `router/router.go`: 添加 chatHandler 参数和路由
- `cmd/server/main.go`: 初始化 ChatService 和 ChatHandler，启动 ChatHub

## 5. 技术要点

### 5.1 WebSocket 协议

**客户端 -> 服务端:**
- `message`: 发送消息
- `stop`: 停止生成
- `ping`: 心跳

**服务端 -> 客户端:**
- `assistant_start`: AI 开始回答
- `thinking_start`: 开始思考
- `thinking_end`: 思考结束
- `tool_call`: 工具调用
- `tool_result`: 工具结果
- `content_delta`: 内容增量
- `assistant_end`: 回答完成
- `stopped`: 已停止
- `error`: 错误
- `pong`: 心跳响应

### 5.2 实时持久化策略

1. 用户消息：立即同步保存
2. AI 消息：创建时保存（status=streaming），流式更新 content
3. 工具调用：开始时保存（status=running），完成后更新结果
4. 最终状态：完成或停止时更新 status

### 5.3 停止机制

- 前端发送 `stop` 消息
- 后端通过 context.Cancel 取消 Agent 执行
- 保留已生成的内容，状态设为 stopped

## 6. 配置参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 单条消息最大长度 | 40000 字符 | 前端限制 |
| 单会话最大消息数 | 1000 条 | 系统限制 |
| WebSocket 鉴权 | Header Bearer Token | 安全验证 |
| 上下文长度限制 | 8000 Token | Agent 参数 |
| 历史消息加载 | 默认 10 条 | 分页加载 |
| 会话过期时间 | 7 天 | 数据保留 |
| 最大重连次数 | 3 次 | 自动重连 |

## 8. 已知限制

1. **Agent 执行**: 目前为框架实现，需要接入实际的 Eino ADK Agent 执行
2. **上下文压缩**: 长对话未做上下文压缩，超过 Token 限制会报错
3. **消息反馈**: 点赞/点踩功能暂未实现数据持久化
4. **会话分享**: 对话分享功能未实现
