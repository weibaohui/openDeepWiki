# DocCopilot 缩放与 Chat 能力集成需求

## 变更记录表

| 版本 | 日期 | 变更内容 | 作者 |
|------|------|----------|------|
| v1.0 | 2025-03-04 | 初始需求文档创建 | AI Assistant |

---

## 1. 需求背景

当前 DocViewer 页面已集成了 DocCopilot AI 助手组件，但存在以下问题：
1. DocCopilot 采用固定宽度的侧边栏形式，显示区域较小，不利于阅读较长的对话内容
2. DocCopilot 的对话能力较为简单，没有集成 ChatPage 中的完整对话功能（如工具调用显示、思考过程展示等）
3. 用户无法根据需要调整 AI 助手界面的显示大小

## 2. 需求目标

1. **支持 DocCopilot 界面缩放**：用户可以将 DocCopilot 从小型侧边栏模式切换为更大的全屏/大屏模式，以便更好地查看对话内容
2. **集成 ChatPage 的对话能力**：将 ChatPage 中成熟的对话功能集成到 DocCopilot 中，包括：
   - 工具调用过程的可视化展示
   - 思考过程的折叠/展开显示
   - 流式消息渲染
   - 消息操作（复制、重试、点赞/踩）
3. **保持界面一致性**：缩放状态的切换应流畅自然，不影响用户体验

## 3. 功能需求

### 3.1 DocCopilot 缩放功能

| 需求编号 | 需求描述 | 优先级 |
|----------|----------|--------|
| DOC-001 | 在 DocCopilot 右上角添加"放大/缩小"切换按钮 | P0 |
| DOC-002 | 点击放大按钮后，DocCopilot 占据更大的屏幕面积（建议宽度 800px 或全屏） | P0 |
| DOC-003 | 点击缩小按钮后，DocCopilot 恢复为小型侧边栏模式（宽度 380px） | P0 |
| DOC-004 | 缩放状态的切换应有平滑的动画过渡效果 | P1 |
| DOC-005 | 放大模式下，应支持点击关闭按钮或再次点击缩小按钮返回小模式 | P0 |

### 3.2 Chat 能力集成

| 需求编号 | 需求描述 | 优先级 |
|----------|----------|--------|
| CHAT-001 | 集成 useChat Hook 的完整功能，包括 WebSocket 连接管理 | P0 |
| CHAT-002 | 支持工具调用过程的可视化展示（如搜索代码、读取文件等） | P0 |
| CHAT-003 | 支持思考过程的显示（<thinking> 标签内容） | P0 |
| CHAT-004 | 支持最终答案的独立展示（<final> 标签内容） | P0 |
| CHAT-005 | 消息应支持复制、重试操作 | P1 |
| CHAT-006 | 支持消息状态的实时更新（发送中、流式接收中、完成） | P0 |
| CHAT-007 | 支持新建对话功能 | P0 |
| CHAT-008 | 保持当前仓库和文档的上下文关联 | P0 |

## 4. 界面设计

### 4.1 小型模式（默认）

```
+---------------+------------------------+---------+
|               |                        |         |
|   文档目录     |      文档内容           |  [放大] |  <- DocCopilot (380px)
|   (250px)     |       (flexible)       |         |
|               |                        |         |
+---------------+------------------------+---------+
```

### 4.2 放大模式

```
+---------------+------------------------+---------------+
|               |                        |               |
|   文档目录     |      文档内容           |   DocCopilot  |  <- 放大模式 (800px+)
|   (250px)     |       (flexible)       |   [缩小]      |
|               |                        |               |
+---------------+------------------------+---------------+
```

或采用覆盖层/弹窗形式：

```
+---------------+------------------------+---------------+
|               |                        |               |
|   文档目录     |      文档内容           |   DocCopilot  |
|   (250px)     |       (flexible)       |   (380px)     |
|               |                        |               |
+---------------+------------------------+---------------+
                          |
                    [放大后覆盖显示]
                          |
+-------------------------+---------------------------+
|                                                   |
|              DocCopilot 放大界面                   |
|              (800px 宽度，居中或右侧固定)            |
|                                                   |
+---------------------------------------------------+
```

## 5. 技术约束

1. **性能要求**：缩放切换不应导致页面重新渲染或数据重新加载
2. **状态保持**：缩放状态切换时，当前对话内容和输入框内容应保持不变
3. **响应式**：放大模式在移动端应降级为全屏模式
4. **WebSocket**：复用现有的 useChat Hook，保持 WebSocket 连接稳定

## 6. 依赖关系

- 依赖 057-AI对话问答功能 提供的 useChat Hook
- 依赖 ChatPage 中的消息渲染组件（MessageContent、MessageFooter 等）
- 需要复用 ChatPage 中的样式定义

## 7. 验收标准

1. [ ] 用户可以点击按钮在大小模式间切换
2. [ ] 放大模式下对话区域显示更宽敞，便于阅读
3. [ ] 工具调用和thinking过程正确显示
4. [ ] 消息可以正常发送和接收
5. [ ] 缩放切换时对话内容不丢失
6. [ ] 移动端适配正常

## 8. 风险说明

1. **布局冲突**：放大模式可能与文档内容区域产生布局冲突，需要仔细处理
2. **性能问题**：放大模式下更多内容的渲染可能影响性能
3. **状态同步**：需要确保缩放状态与对话状态正确同步
