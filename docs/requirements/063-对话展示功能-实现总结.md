## 1. 实现概述

本功能实现了对话展示系统，允许用户将 AI 对话设为公开，供其他用户查看和学习。

### 1.1 核心功能

1. **对话公开/私有设置** - 在 DocCopilot 中一键控制对话可见性
2. **对话列表页面** - 展示仓库下所有公开对话
3. **对话详情查看** - 查看完整对话内容，包括思考过程和工具调用
4. **集成导航** - DocViewer 左侧菜单新增"对话记录"入口

#### 2.2.3 Hook 扩展

**文件**: `frontend/src/hooks/useChat.ts`

新增 `updateLocalSession` 方法，支持乐观更新策略：
```typescript
const updateLocalSession = useCallback((sessionId: string, updates: Partial<ChatSession>) => {
  setState((prev) => {
    const updatedSessions = prev.sessions.map((s) =>
      s.session_id === sessionId ? { ...s, ...updates } : s
    );
    const updatedCurrentSession =
      prev.currentSession?.session_id === sessionId
        ? { ...prev.currentSession, ...updates }
        : prev.currentSession;
    return {
      ...prev,
      sessions: updatedSessions,
      currentSession: updatedCurrentSession,
    };
  });
}, []);
```

## 3. 关键设计决策

### 3.1 乐观更新策略

**问题**：如果等待 API 响应后再更新 UI，用户会感觉卡顿。

**决策**：采用乐观更新，先更新 UI 状态，再调用 API，失败时回滚。

**实现**：
```typescript
// 1. 立即更新本地状态
updateLocalSession(sessionId, { visibility: 'public' });

try {
  // 2. 调用 API
  await chatApi.updateVisibility(repoId, sessionId, 'public');
} catch {
  // 3. 失败时回滚
  updateLocalSession(sessionId, { visibility: 'private' });
}
```

### 3.2 默认私有策略

**决策**：新创建的对话默认为私有，需要用户主动设为公开。

**理由**：
- 保护用户隐私
- 避免敏感对话意外泄露
- 让用户有控制权

### 3.3 文档列表默认展开

**决策**：DocViewer 左侧「文档列表」分组默认展开。

**理由**：
- 减少用户点击次数
- 文档是对话场景的主要上下文
- 与对话列表并列展示，方便切换
