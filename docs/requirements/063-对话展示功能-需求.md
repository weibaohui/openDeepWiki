## 1. 需求背景

当前系统已实现了 DocCopilot AI 对话功能，用户可以针对仓库代码进行问答对话。但存在以下问题：

1. **对话无法被其他人查看**：对话内容仅对参与者可见，其他用户无法浏览和学习
2. **知识无法沉淀**：有价值的问答对话无法像文档一样被积累和分享
3. **缺乏对话管理入口**：用户无法像管理文档一样管理历史对话

为了让对话内容能够像文档一样被展示和分享，需要实现一个对话展示功能。

---

## 2. 功能目标

### 2.1 核心目标

1. **对话列表展示**：在仓库下展示所有历史对话列表
2. **对话详情查看**：支持查看完整对话内容（包括思考过程、工具调用等）
3. **对话公开性**：对话可设置为公开或私有，公开的对话可被其他用户查看
4. **类似文档的体验**：对话展示体验与文档展示保持一致

### 2.2 功能边界

**包含范围：**
- 对话列表页面（类似文档列表）
- 对话详情展示页面（类似文档详情）
- 对话的公开/私有设置
- 在 DocViewer 中集成对话入口

**不包含范围：**
- 对话编辑功能（对话一旦生成不可修改）
- 对话评论功能（后续迭代考虑）
- 对话点赞/收藏统计（后续迭代考虑）

## 3. 用户故事

### 3.1 故事一：浏览对话列表

作为用户，我希望在仓库页面看到所有公开的对话列表，这样我可以了解其他人在讨论什么问题。

**验收标准：**
- 在仓库左侧导航增加"对话"入口
- 点击后显示该仓库下的所有公开对话列表
- 列表显示对话标题、创建时间、消息数量
- 支持按时间排序

### 3.2 故事二：查看对话详情

作为用户，我希望点击某个对话后可以查看完整的对话过程，包括 AI 的思考过程和工具调用。

**验收标准：**
- 对话详情页展示完整的问答过程
- 展示用户的提问和 AI 的回答
- 展示 AI 的思考过程（可折叠）
- 展示工具调用详情（工具名、参数、结果）
- 界面风格与 DocViewer 保持一致

### 3.3 故事三：设置对话公开性

作为对话创建者，我希望可以设置对话是公开还是私有，保护敏感对话内容。

**验收标准：**
- 在 DocCopilot 中可以设置对话公开/私有
- 私有对话仅创建者可见
- 公开对话所有用户可见
- 默认创建为私有对话

### 3.4 故事四：从文档页面进入对话

作为用户，我希望在查看文档时可以方便地查看相关的对话记录。

**验收标准：**
- DocViewer 左侧导航增加"对话"标签
- 显示该仓库下的公开对话列表
- 点击对话可在右侧或新页面打开

## 4. 详细需求说明

### 4.1 数据模型扩展

#### 4.1.1 扩展 chat_sessions 表

```go
type ChatSession struct {
    ID            uint           `gorm:"primaryKey"`
    SessionID     string         `gorm:"size:64;uniqueIndex;not null"`
    RepositoryID  uint           `gorm:"index;not null"`
    Title         string         `gorm:"size:255"`
    Status        string         `gorm:"size:20;default:'active'"`
    Visibility    string         `gorm:"size:20;default:'private'"`  // 新增：public, private
    CreatedBy     uint           `gorm:"index;default:0"`            // 新增：创建者用户ID
    MessageCount  int            `gorm:"default:0"`                  // 新增：消息数量缓存
    CreatedAt     time.Time
    UpdatedAt     time.Time
}
```

## 5. 技术实现要点

### 5.1 后端实现

1. **数据库迁移**
   - chat_sessions 表增加 visibility、created_by、message_count 字段

2. **权限控制**
   - 公开对话：所有用户可查看
   - 私有对话：仅创建者可查看
   - 更新可见性：仅创建者可修改

3. **API 实现**
   - 复用现有 ChatService
   - 新增公开列表查询（只返回 visibility='public' 的会话）
   - 新增详情查看接口（增加权限校验）

### 5.2 前端实现

1. **页面组件**
   - ChatSessionListPage：对话列表页面
   - ChatSessionViewer：对话详情展示组件
   - ChatSessionList：对话列表组件（复用到 DocViewer 侧边栏）

2. **路由配置**
   - `/repo/:id/chat` - 对话列表
   - `/repo/:id/chat/:sessionId` - 对话详情

3. **样式复用**
   - 复用 DocViewer 的布局和样式
   - 复用 DocCopilot 的消息渲染组件

## 6. 验收标准

### 6.1 功能验收

- [ ] 左侧导航增加"对话"入口
- [ ] 对话列表页面显示所有公开对话
- [ ] 点击对话可查看完整对话详情
- [ ] 对话详情展示思考过程和工具调用
- [ ] DocCopilot 可设置对话公开/私有
- [ ] 私有对话仅创建者可见
- [ ] 公开对话所有用户可见

### 6.2 性能验收

- [ ] 对话列表加载时间 < 1 秒
- [ ] 对话详情加载时间 < 2 秒
- [ ] 消息较多时分页加载

### 6.3 安全验收

- [ ] 私有对话无法被其他用户访问
- [ ] 权限校验在每个 API 端点生效
