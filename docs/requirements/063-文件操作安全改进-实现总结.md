## 1. 需求对应

本实现对应需求：**文件获取的安全改进**

限制对文件操作的目录，所有读写文件、获取文件夹目录等操作，只能在本代码仓库的本地目录地址内进行，不能超出这个范围。

## 2. 实现概述

### 2.1 核心改进

#### 1. 增强路径安全验证 (`path_utils.go`)

- **符号链接解析**：新增 `resolvePathWithSymlinks` 函数，递归解析路径中的符号链接，防止通过符号链接访问基础目录之外的文件
- **路径遍历防护**：正确处理 `../` 等路径遍历序列，阻止访问父目录
- **目录边界精确匹配**：使用路径分隔符确保精确匹配目录边界，防止 `/safe/base` 和 `/safe/base_backup` 这类前缀相似目录被误判

- `ValidateWorkingDir`：工作目录验证函数

#### 3. 终端命令工具加固 (`cmd.go`)

- **工作目录强制限制**：执行命令前验证 `working_dir` 必须在 `basePath` 范围内
- **命令安全检查**：检测并阻止包含路径遍历序列（`../`、`..\`）的命令
- **新增 `basePath` 字段**：保存基础路径用于安全验证

#### 4. Git Clone 工具加固 (`git_clone.go`)

- **目标目录安全检查**：验证 `target_dir` 参数不包含路径遍历序列

## 3. 关键实现点

### 3.1 符号链接处理

```go
// resolvePathWithSymlinks 解析路径中的符号链接
// 如果路径不存在，尝试解析其存在的父目录
func resolvePathWithSymlinks(path string) string {
    // 首先尝试直接解析
    resolved, err := filepath.EvalSymlinks(path)
    if err == nil {
        return filepath.Clean(resolved)
    }

    // 如果路径不存在，逐步向上查找存在的目录并解析
    // 然后拼接剩余部分
    dir := path
    var unresolvedParts []string

    for {
        parent := filepath.Dir(dir)
        if parent == dir {
            break
        }
        unresolvedParts = append([]string{filepath.Base(dir)}, unresolvedParts...)
        resolvedParent, err := filepath.EvalSymlinks(parent)
        if err == nil {
            result := resolvedParent
            for _, part := range unresolvedParts[1:] {
                result = filepath.Join(result, part)
            }
            return filepath.Clean(result)
        }
        dir = parent
    }

    return filepath.Clean(path)
}
```

## 6. 已知限制与注意事项

1. **符号链接处理**：对于指向不存在的目标文件的符号链接，可能无法完全解析其最终路径。但在实际文件操作时，操作系统会再次检查。

2. **终端命令限制**：当前仅检测明显的路径遍历序列（`../`、`..\`），复杂的命令注入可能需要更严格的命令白名单机制。

3. **性能考虑**：符号链接解析需要额外的系统调用，但对于文件操作场景，这种开销是可以接受的。
