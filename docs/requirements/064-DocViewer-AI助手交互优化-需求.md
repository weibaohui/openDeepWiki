## 1. 需求背景

当前 DocViewer 页面存在以下用户体验问题：

1. **文档显示区域边界不清晰**：文档内容与背景色相同，用户难以区分文档边界
2. **标题栏拥挤**：当 AI 助手打开时，顶部标题栏按钮过多，空间拥挤
3. **标题栏高度不一致**：左侧文档标题栏与右侧 AI 助手标题栏高度不同，视觉不协调
4. **AI助手展开体验不佳**：点击"更大"后，AI 助手宽度固定，未充分利用屏幕空间
5. **关闭AI助手后布局未恢复**：关闭 AI 助手后，文档显示区域未自动恢复

## 2. 需求目标

优化 DocViewer 页面的交互体验，使文档显示与 AI 助手协作更加流畅、美观。

## 3. 功能需求

### 3.1 文档显示区域背景优化

**需求描述**：
- 为文档显示区域添加白色背景（或主题对应背景色）
- 与外层灰色背景形成明显区分

**验收标准**：
- 文档内容区域有清晰的边界
- 支持暗黑主题，背景色自动切换

### 3.2 功能按钮位置优化

**需求描述**：
- 将导出、编辑、保存、重新生成、版本等功能按钮从顶部标题栏移除
- 移至文档内容区域内部
- 放在"创建于/更新于"行的右侧
- 使用下拉菜单收拢这些按钮

**验收标准**：
- 标题栏仅保留 AI Copilot 按钮
- 文档操作按钮在 meta 信息行右侧
- 使用小图标、小字号
- 响应式：AI助手打开时隐藏操作菜单（空间不足）

### 3.3 标题栏高度统一

**需求描述**：
- 左侧文档标题栏高度与右侧 AI 助手标题栏高度一致
- 统一为 52px

**验收标准**：
- 两个标题栏高度相同
- 视觉对齐

### 3.4 AI助手展开全屏优化

**需求描述**：
- 点击"更大"按钮后，AI 助手展开占据全部剩余空间
- 隐藏左侧文档显示区域
- 对话内容居中显示，两边留小空白
- 最大内容宽度 900px，扩大显示区域

**验收标准**：
- 文档区域完全隐藏
- AI 助手占据左侧目录右侧到页面边缘的全部空间
- 对话列表、欢迎语、输入框居中显示
- 内容最大宽度 900px，自适应屏幕

### 3.5 关闭AI助手恢复布局

**需求描述**：
- 点击 AI 助手关闭按钮（X）后
- 自动恢复文档显示区域
- 重置 AI 助手展开状态

**验收标准**：
- 关闭 AI 助手后，文档区域立即显示
- AI 助手状态重置为收起状态

## 4. 非功能需求

| 项目 | 要求 |
|------|------|
| 性能 | 布局切换动画流畅，无卡顿 |
| 兼容性 | 支持响应式，适配不同屏幕尺寸 |
| 可访问性 | 按钮有清晰的 hover/active 状态 |

## 5. 界面原型

### 5.1 正常状态（AI助手收起）
```
+----------------------------------+----------+
| 文档标题                    [AI] | AI助手   |
| 创建于... 更新于... [操作▼]       | 标题栏   |
+----------------------------------+          |
|                                  |          |
| 文档内容（白色背景）              |          |
|                                  |          |
|                                  |          |
+----------------------------------+----------+
```

### 5.2 展开状态（AI助手放大）
```
+----------+------------------------------------------+
| 文档列表 | AI助手标题栏                        [X]  |
|          +------------------------------------------+
|          |                                          |
|          |    对话内容（居中，最大宽度900px）        |
|          |                                          |
|          |    [输入框]                              |
|          |                                          |
+----------+------------------------------------------+
```

## 6. 技术要点

- 使用 CSS 变量 `var(--ant-color-bg-container)` 支持主题切换
- 使用 Flex 布局实现自适应
- 使用 `useEffect` 监听状态变化
- 组件间通过 props 回调传递状态
