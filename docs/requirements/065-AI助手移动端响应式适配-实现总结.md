## 3. 关键实现点

### 3.1 DocViewer.tsx 修改

#### 状态管理
```typescript
const [copilotOpen, setCopilotOpen] = useState(false);
const [copilotExpanded, setCopilotExpanded] = useState(false);
```

#### 点击处理逻辑
```typescript
onClick={() => {
    setCopilotOpen(true);
    // 小屏下自动展开AI助手（全屏模式）
    if (!screens.md) {
        setCopilotExpanded(true);
    }
}}
```

#### AI 助手容器响应式样式
```typescript
<div style={{
    display: 'flex',
    flexDirection: 'column',
    height: '100vh',
    // 小屏下始终全屏，大屏下根据展开状态
    flex: (copilotExpanded || !screens.md) ? 1 : 'unset',
    width: (copilotExpanded || !screens.md) ? '100%' : undefined,
    position: !screens.md ? 'absolute' : undefined,
    top: !screens.md ? 0 : undefined,
    left: !screens.md ? 0 : undefined,
    zIndex: !screens.md ? 100 : undefined,
}}>
```

#### 状态通知优化
修复组件挂载时立即触发 `onExpandChange` 的问题：
```typescript
const prevExpandedRef = useRef(isExpanded);
useEffect(() => {
    if (prevExpandedRef.current !== isExpanded) {
        prevExpandedRef.current = isExpanded;
        onExpandChange?.(isExpanded);
    }
}, [isExpanded, onExpandChange]);
```

## 4. 技术难点与解决

### 4.1 状态重置问题

**现象**：点击 AI 按钮后，AI 助手闪了一下就消失了。

**原因**：`DocCopilot` 组件挂载时的 `useEffect` 立即调用 `onExpandChange(false)`，导致父组件关闭 AI 助手。

**解决**：添加 `prevExpandedRef` 跟踪之前的状态，仅在状态真正变化时才通知父组件。

### 4.2 响应式初始化

**现象**：`useBreakpoint` 初始返回 `undefined`，导致初始状态判断错误。

**解决**：默认关闭 AI 助手，等待屏幕尺寸确定后由用户主动打开。
