## 1. 需求背景

当前 DocViewer 页面的 AI 助手在移动端存在以下用户体验问题：

1. **小屏设备空间占用过大**：AI 助手默认打开，在移动端占用 380px 宽度，导致文档内容显示区域过小
2. **分屏模式不合理**：移动端点击 AI 按钮后，AI 助手与文档分屏显示，可用空间极小
3. **操作体验差**：移动端用户更期望全屏对话体验，而非狭窄的侧边栏

## 2. 需求目标

优化 AI 助手在移动端的显示逻辑，使其更适合小屏设备的使用场景，同时保持桌面端的良好体验。

## 3. 功能需求

### 3.1 小屏默认隐藏 AI 助手

**需求描述**：
- 在移动设备（屏幕宽度 < 768px）上，默认关闭 AI 助手面板
- 只显示 AI 助手按钮（机器人图标）
- 桌面端（屏幕宽度 >= 768px）保持默认打开 AI 助手

**验收标准**：
- 移动端首次加载时 AI 助手处于关闭状态
- 仅显示机器人图标按钮
- 桌面端不受影响，保持默认打开

### 3.2 小屏全屏显示 AI 助手

**需求描述**：
- 移动端点击 AI 助手按钮后，AI 助手占满整个屏幕空间
- 使用绝对定位覆盖整个视口（position: absolute, top: 0, left: 0, zIndex: 100）
- 文档内容区域完全隐藏
- 桌面端保持原有展开/收缩行为

**验收标准**：
- 移动端 AI 助手宽度为 100%
- 高度为 100vh
- 文档区域不可见
- 可正常关闭 AI 助手返回文档

### 3.3 响应式断点

**需求描述**：
- 使用 Ant Design 的 useBreakpoint 检测屏幕尺寸
- 断点：md（768px）
- 小屏：screens.md === false
- 大屏：screens.md === true

**验收标准**：
- 响应式检测正常工作
- 窗口大小变化时正确切换模式

## 4. 非功能需求

| 项目 | 要求 |
|------|------|
| 性能 | 布局切换动画流畅，无卡顿 |
| 兼容性 | 支持主流移动设备（iOS Safari、Android Chrome） |
| 可访问性 | 按钮有清晰的点击反馈 |

## 5. 界面原型

### 5.1 移动端初始状态（AI助手关闭）
```
+------------------+
| ≡  文档概览   🤖 |  <- 机器人按钮
+------------------+
|                  |
|   文档内容       |
|   （全屏显示）   |
|                  |
+------------------+
```

### 5.2 移动端 AI 助手打开状态（全屏）
```
+------------------+
| ● AI助手    + ☐ ✕|  <- 全屏显示
+------------------+
|                  |
|  👋 AI 文档助手   |
|  基于当前文档...  |
|                  |
|  [输入消息...]   |
+------------------+
```

### 5.3 桌面端状态（保持原有行为）
```
+------------------+----------+
| 文档标题    [AI] | AI助手   |  <- 默认打开
+------------------+          |
|                  |          |
| 文档内容         |          |
|                  |          |
+------------------+----------+
```

## 6. 技术要点

- 使用 Ant Design 的 `useBreakpoint` 进行响应式检测
- 使用绝对定位实现移动端全屏覆盖
- 通过 props 回调控制 AI 助手显示/隐藏状态
- 小屏下移除展开/收缩按钮，简化界面
