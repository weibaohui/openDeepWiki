# 067-Agents资源嵌入二进制-需求

## 1. 问题背景

### 1.1 当前问题
用户直接下载预编译的二进制文件（如 `opendeepwiki-linux-amd64`）在服务器运行时，程序报错：

```
Agent 执行出错: agent 为空
```

### 1.2 根本原因
程序依赖 `backend/agents/` 目录下的 YAML 配置文件来定义 AI Agent 的行为。这些文件在源码仓库中存在，但未包含在预编译的二进制发布包中，导致用户下载二进制后无法正常运行。

### 1.3 影响范围
- 从 GitHub Releases 下载二进制文件的用户
- 使用 Docker 部署但未挂载 volumes 的用户
- 所有需要独立部署的场景

## 2. 需求目标

### 2.1 主要目标
将 Agents 配置文件嵌入到二进制文件中，使程序在首次启动时能够自动释放所需的默认配置文件，实现"单文件部署"。

### 2.2 具体需求

#### R1: Agents 资源嵌入
- 使用 Go 的 `embed` 特性将 `backend/agents/` 目录下的所有 YAML 文件打包进二进制
- 嵌入的文件应包含 13 个默认 Agent 定义文件

#### R2: 自动释放机制
- 程序启动时检测 `agents/` 目录是否存在
- 如目录不存在，自动创建目录并释放内嵌的默认 Agent 文件
- 如目录已存在且有文件，不覆盖用户自定义的 Agent 配置

#### R3: Skills 目录自动创建
- 程序启动时检测 `skills/` 目录是否存在
- 如目录不存在，自动创建空目录
- Skills 内容不由程序提供，用户可自行添加

#### R4: 构建流程更新
- Makefile 中添加 agents 嵌入的预处理步骤
- 编译前自动将 agents 文件复制到嵌入目录
- 编译后清理临时文件，保持仓库整洁

#### R5: 向后兼容
- 对已存在的用户配置无影响
- 不破坏现有的 Agent 热加载机制

## 3. 非功能需求

### 3.1 性能要求
- 启动时的文件释放应在 100ms 内完成
- 不增加二进制文件显著大小（当前 agents 约 80KB）

### 3.2 可靠性
- 释放过程应保证原子性
- 错误处理应明确，失败时给出清晰错误信息

## 4. 验收标准

| 编号 | 验收项 | 验证方式 |
|------|--------|----------|
| AC1 | 下载二进制到新目录，直接运行，程序正常启动 | 手动测试 |
| AC2 | 运行后检查 `./agents/` 目录，包含 13 个 YAML 文件 | 手动测试 |
| AC3 | 运行后检查 `./skills/` 目录，目录存在且为空 | 手动测试 |
| AC4 | 修改某个 agent 文件后重启，修改被保留 | 手动测试 |
| AC5 | `make build` 成功生成包含 agents 的二进制 | 自动化 |
