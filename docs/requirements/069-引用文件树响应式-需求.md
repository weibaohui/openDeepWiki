## 1. 背景（Why）

在 DocViewer 文档显示页面中，右侧存在一个"引用文件"树组件，用于展示当前文档中引用的所有源文件路径。在大屏幕（xl 及以上）时，该树组件以 sticky 侧边栏形式展示，宽度固定 320px。

**当前问题**：
- 在中等屏幕（如 lg、md）下，引用文件树与文档内容并行显示，占用额外空间，导致文档阅读区域被压缩
- 在小屏幕（xs、sm）下，引用文件树虽然已改为 column 布局（宽度 100%），但仍然始终展开显示，占用大量屏幕空间
- 用户在移动端或窄屏环境下，需要频繁滚动才能看到文档正文内容

## 2. 目标（What，必须可验证）

- [ ] **O1**: 在 screens.xl 以下断点，引用文件树默认折叠收起，仅显示标题栏
- [ ] **O2**: 引用文件树支持用户手动展开/折叠操作
- [ ] **O3**: 引用文件树折叠状态在同会话内保持（可选，视复杂度而定）
- [ ] **O4**: 展开/折叠操作有平滑过渡动画
- [ ] **O5**: 折叠后显示展开按钮，展开后显示折叠按钮
- [ ] **O6**: AI 助手打开时隐藏引用文件树区域，避免正文区域拥挤

## 3. 非目标（Explicitly Out of Scope）

- 不修改引用文件树的内部数据结构（`ReferenceTreeNode`）
- 不修改引用文件的提取逻辑（`extractReferenceLinks`、`buildReferenceTreeData`）
- 不修改引用文件的点击跳转逻辑（`handleReferenceTreeSelect`）
- 不修改 AI Copilot 的展开/折叠行为（那是另一个需求）

## 4. 使用场景 / 用户路径

### 场景 1：桌面端用户查看引用文件
- 用户在大屏幕（xl+）下查看文档
- 引用文件树以 sticky 侧边栏形式始终可见
- 用户可以直接浏览引用的文件路径结构

### 场景 2：移动端用户专注阅读文档
- 用户在手机或平板（lg 以下）上查看文档
- 引用文件树默认折叠，仅显示"引用文件"标题和展开按钮
- 用户点击"展开"按钮可查看引用文件
- 用户再次点击"折叠"按钮可收起引用文件树

### 场景 3：中等屏幕用户偶尔查看引用
- 用户在中等屏幕（lg、md）上查看文档
- 引用文件树默认折叠
- 用户需要时手动展开，查看完可再次折叠

### 场景 4：AI 助手与文档协同阅读
- 用户打开 AI 助手后，页面右侧出现 AI 助手区域
- 引用文件树区域自动隐藏，让正文区域保留更多可读宽度
- 用户关闭 AI 助手后，引用文件树恢复显示

## 5. 功能需求清单（Checklist）

- [ ] **F1**: 引用文件树组件添加折叠/展开状态管理（`referenceTreeCollapsed` state）
- [ ] **F2**: 引用文件树 Card 组件添加折叠按钮（ChevronRight / ChevronDown 图标）
- [ ] **F3**: 引用文件树内容区域根据折叠状态条件渲染（`collapsed ? null : <Tree />`）
- [ ] **F4**: 折叠状态使用 CSS transition 实现平滑过渡（height collapse）
- [ ] **F5**: screens.xl 以下断点默认折叠（通过 useEffect 或 useMemo 设置初始值）
- [ ] **F6**: 添加国际化文案 key（`document.reference_tree_collapse`、`document.reference_tree_expand`）
- [ ] **F7**: 当 `copilotOpen=true` 时，不渲染引用文件树区域

## 6. 约束条件

### 6.1 技术约束
- 使用 React Hooks（useState）管理折叠状态
- 使用 Ant Design 的 Collapse 机制或自定义实现
- 折叠动画时长建议 200-300ms

### 6.2 响应式断点
- `screens.xl`（≥1200px）：保持现有行为，引用文件树始终展开显示在右侧
- `screens.lg`（≥992px）以下：引用文件树默认折叠
- `screens.md`（≥768px）以下：引用文件树占满整行宽度（已有）

### 6.3 样式约束
- 折叠时 Card 保持最小高度，仅显示标题栏
- 展开按钮样式与页面其他按钮风格一致
- 不破坏现有的 sticky 定位效果

## 7. 可修改 / 不可修改项

- ❌ 不可修改：
  - `extractReferenceLinks` 函数逻辑
  - `buildReferenceTreeData` 函数逻辑
  - `ReferenceTreeNode` 接口结构
  - `handleReferenceTreeSelect` 点击处理逻辑
  - 现有的布局结构（Sider、Content、DocCopilot）

- ✅ 可调整：
  - 引用文件树 Card 的样式和布局
  - 折叠状态的管理方式
  - 折叠/展开按钮的图标和位置
  - 折叠动画的实现方式

## 8. 接口与数据约定（如适用）

本需求不涉及接口变更，仅涉及前端状态管理和 UI 交互。

## 9. 验收标准（Acceptance Criteria）

| 编号 | 验收项 | 验证方式 |
|------|--------|----------|
| AC1 | 在 lg 断点下，引用文件树默认折叠 | 手动测试：调整浏览器宽度至 lg 断点，刷新页面 |
| AC2 | 在 xl 断点下，引用文件树默认展开 | 手动测试：调整浏览器宽度至 xl 断点，刷新页面 |
| AC3 | 点击展开按钮后，引用文件树内容正确显示 | 手动测试：点击展开按钮，验证 Tree 组件渲染 |
| AC4 | 点击折叠按钮后，引用文件树内容正确隐藏 | 手动测试：点击折叠按钮，验证内容消失 |
| AC5 | 展开/折叠有平滑过渡动画 | 手动测试：观察动画效果 |
| AC6 | 折叠状态下 Card 高度最小化，不遮挡文档内容 | 手动测试：窄屏下滚动查看文档 |
| AC7 | AI 助手打开时，引用文件树区域隐藏 | 手动测试：打开 AI 助手，验证引用文件树区域不渲染 |
| AC8 | AI 助手关闭后，引用文件树区域恢复显示 | 手动测试：关闭 AI 助手，验证引用文件树恢复 |

## 10. 风险与已知不确定点

| 风险点 | 描述 | 应对方案 |
|--------|------|----------|
| 折叠动画与 Card 组件原生动画冲突 | Ant Design Card 可能有默认动画 | 使用 CSS transition 控制内部容器高度，而非 Card 本身 |
| 初始状态与断点变化时的状态同步 | 窗口从 xl 缩小到 lg 时，当前展开的树应立即折叠 | 使用 `useEffect` 监听 `screens` 变化，强制同步折叠状态 |
| 国际化文案缺失 | 目前没有这些文案的 key | 新增 `document.reference_tree_collapse` 和 `document.reference_tree_expand` |
| AI 助手显示与引用树互斥逻辑遗漏 | 需求文档未覆盖该条件会导致回归测试遗漏 | 在目标、清单、验收标准中明确 `copilotOpen` 相关行为 |
