## 1. 实现了什么

在文档查看页面（DocViewer）右侧新增了「目录（TOC）」面板，与「引用文件」通过 Ant Design Tabs 组件并列展示。

---

## 2. 与需求的对应关系

| 需求 | 实现 |
|------|------|
| 自动解析文档标题（h1~h3） | `parseHeadings()` 函数，正则匹配 `^(#{1,3})\s+(.+)$` |
| 层级缩进展示 | `DocToc` 组件，`paddingLeft = (level - 1) * 12 + 8` px |
| 点击跳转 | `scrollIntoView({ behavior: 'smooth' })` |
| 当前章节高亮 | `IntersectionObserver`，`rootMargin: '-10% 0px -80% 0px'` |
| 无标题时隐藏 | `hasToc = tocHeadings.length > 0`，控制 `showRightPanel` |
| 与引用文件共存 | Ant Design `Tabs` 组件，两者共用一个右侧面板 |
| AI 助手展开时隐藏 | `showRightPanel = ... && !copilotOpen` |
| 移动端不显示 | 继承原有逻辑（`screens.xl` 控制右侧面板仅在大屏展示） |

---

## 3. 关键实现点

### 3.1 新增文件

- `frontend/src/components/DocToc.tsx`：TOC 组件，包含 `slugify`、`parseHeadings` 两个可导出工具函数

### 3.2 修改文件

- `frontend/src/components/markdown/MarkdownRender.tsx`：
  - 导入 `slugify`
  - 新增 `createHeadingComponent` 工厂函数，为 h1/h2/h3 添加 `id` 属性
  - 将 `h1/h2/h3` 自定义组件传入 `XMarkdown` 的 `components` prop

- `frontend/src/pages/DocViewer.tsx`：
  - 导入 `DocToc`、`parseHeadings`、`TocHeading`、`Tabs`
  - 新增 `tocHeadings` useMemo（解析当前文档标题）
  - 新增 `hasToc` 和 `showRightPanel` 计算属性
  - 移除引用文件树的折叠状态（`referenceTreeCollapsed`、`toggleReferenceTree`）及相关 effect
  - 右侧面板改为 Tabs 布局，默认显示「目录」Tab

### 3.3 id 一致性

`parseHeadings`（用于 TOC 列表）和 `createHeadingComponent`（用于渲染时添加 id）均使用同一个 `slugify` 函数，保证目录中的 id 与 DOM 中的 id 完全一致，点击跳转准确。

---

## 4. 已知限制或待改进点

- **重复标题**：当同一文档中出现相同文本的标题时，`parseHeadings` 会为第二个生成 `id-1`，但 `createHeadingComponent` 目前不做计数，会生成相同 `id`。实际文档中重复标题极少见，后续可按需优化
- **HashRouter 冲突**：使用 `scrollIntoView` 而非 URL hash，避免与 React Router HashRouter 的 `#` 冲突，但也意味着刷新页面不会恢复滚动位置
- **XMarkdown components 支持**：`h1/h2/h3` 作为 components key 在构建阶段可通过，但若底层库更新后不支持，需回退为 useEffect + querySelectorAll 方案注入 id
