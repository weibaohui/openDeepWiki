# openDeepWiki 前端自动化测试规范

## 文档概述

本文档定义了 openDeepWiki 项目的 **前端自动化测试规范**，包括测试原则、工具选型、测试框架设计、编码规范和 AI 生成测试指南。

---

## 一、测试原则

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **可预测性** | 测试结果应当稳定、可重复，避免 flaky 测试 |
| **自文档化** | 测试用例应当清晰描述被测功能，便于 AI 理解和生成 |
| **模块化** | 测试应当按功能模块组织，便于 AI 增量编写 |
| **隔离性** | 每个测试应当独立运行，避免依赖其他测试 |
| **快速反馈** | 单元测试应当快速运行（< 1 秒），E2E 测试可以较慢 |
| **可维护性** | 测试代码应当易于更新和扩展 |

### 1.2 测试金字塔

```
        /\
       /E2E\          10% - 关键用户流程
      /------\
     /集成测试\       30% - API 交互、组件集成
    /----------\
   /  单元测试  \     60% - 组件、Hooks、工具函数
  /--------------\
```

### 2.1 测试框架选型

| 类型 | 推荐工具 | 版本建议 | 理由 |
|------|---------|---------|------|
| 单元测试 | Vitest | ^1.x | Vite 原生支持，速度快 |
| 组件测试 | React Testing Library | ^16.x | React 官方推荐 |
| E2E 测试 | Playwright | ^1.45.x | 跨浏览器、支持多语言 |
| 测试覆盖率 | c8 (Vitest 内置) | - | 与 Vitest 集成良好 |
| Mock 工具 | MSW (Mock Service Worker) | ^2.x | 拦截网络请求，真实模拟 |
| 测试辅助 | userEvent | ^14.x | 模拟真实用户交互 |

### 5.1 单元测试规范

#### 5.1.1 组件测试模板

```typescript
/**
 * 组件测试模板
 * AI 应当遵循此模板编写组件测试
 */

import { describe, it, expect, vi, beforeEach } from 'vitest'
import { render, screen, within, waitFor } from '@testing-library/react'
import userEvent from '@testing-library/user-event'
import ComponentName from './ComponentName'

// Mock 依赖
vi.mock('../services/api', () => ({
  apiService: {
    method: vi.fn().mockResolvedValue({ data: [] }),
  },
}))

describe('ComponentName', () => {
  beforeEach(() => {
    vi.clearAllMocks()
  })

  // 1. 渲染测试
  describe('渲染测试', () => {
    it('应该正确渲染默认状态', () => {
      render(<ComponentName />)
      expect(screen.getByRole('button')).toBeInTheDocument()
    })

    it('应该正确接收 props 并渲染', () => {
      render(<ComponentName title="测试标题" />)
      expect(screen.getByText('测试标题')).toBeInTheDocument()
    })
  })

  // 2. 用户交互测试
  describe('交互测试', () => {
    it('点击按钮应该触发 onClick 回调', async () => {
      const handleClick = vi.fn()
      const user = userEvent.setup()

      render(<ComponentName onClick={handleClick} />)

      await user.click(screen.getByRole('button'))
      expect(handleClick).toHaveBeenCalledTimes(1)
    })
  })

  // 3. 边界情况测试
  describe('边界情况', () => {
    it('当 prop 为 null 时不应该崩溃', () => {
      render(<ComponentName data={null} />)
      expect(screen.queryByText('错误')).not.toBeInTheDocument()
    })

    it('当加载数据时应该显示 loading 状态', () => {
      render(<ComponentName loading={true} />)
      expect(screen.getByTestId('loading')).toBeInTheDocument()
    })
  })

  // 4. 状态变化测试
  describe('状态测试', () => {
    it('数据加载成功后应该显示内容', async () => {
      render(<ComponentName />)

      await waitFor(() => {
        expect(screen.getByText('成功')).toBeInTheDocument()
      })
    })

    it('数据加载失败时应该显示错误信息', async () => {
      // Mock API 返回错误
      const { apiService } = await import('../services/api')
      vi.mocked(apiService.method).mockRejectedValue(new Error('API 错误'))

      render(<ComponentName />)

      await waitFor(() => {
        expect(screen.getByText(/错误/i)).toBeInTheDocument()
      })
    })
  })
})
```

#### 5.1.2 Hooks 测试模板

```typescript
/**
 * Hooks 测试模板
 */

import { describe, it, expect, beforeEach } from 'vitest'
import { renderHook, act, waitFor } from '@testing-library/react'
import { useCustomHook } from './useCustomHook'

describe('useCustomHook', () => {
  beforeEach(() => {
    vi.clearAllMocks()
  })

  it('应该返回初始状态', () => {
    const { result } = renderHook(() => useCustomHook())
    expect(result.current.state).toBe('initial')
  })

  it('调用 action 应该更新状态', async () => {
    const { result } = renderHook(() => useCustomHook())
    const { updateState } = result.current

    await act(async () => {
      await updateState('new state')
    })

    expect(result.current.state).toBe('new state')
  })

  it('应该正确处理异步操作', async () => {
    const { result } = renderHook(() => useCustomHook())
    const { fetchData } = result.current

    await act(async () => {
      await fetchData()
    })

    await waitFor(() => {
      expect(result.current.loading).toBe(false)
      expect(result.current.data).toBeDefined()
    })
  })
})
```

### 5.2 API 服务测试规范

```typescript
/**
 * API 服务测试模板
 */

import { describe, it, expect, beforeEach, afterEach } from 'vitest'
import { http } from 'msw'
import { setupServer } from 'msw/node'
import { apiService } from './api'

// Mock 服务器
const server = setupServer()

beforeAll(() => server.listen())
afterEach(() => server.resetHandlers())
afterAll(() => server.close())

describe('apiService', () => {
  describe('getData', () => {
    it('应该成功获取数据', async () => {
      const mockData = { id: 1, name: 'test' }

      server.use(
        http.get('/api/endpoint', (req, res, ctx) => {
          return res(
            ctx.status(200),
            ctx.json({ data: mockData })
          )
        })
      )

      const response = await apiService.getData()

      expect(response.status).toBe(200)
      expect(response.data).toEqual(mockData)
    })

    it('当 API 返回错误时应该抛出异常', async () => {
      server.use(
        http.get('/api/endpoint', (req, res, ctx) => {
          return res(
            ctx.status(500),
            ctx.json({ error: 'Server error' })
          )
        })
      )

      await expect(apiService.getData()).rejects.toThrow()
    })

    it('应该正确处理网络错误', async () => {
      server.use(
        http.get('/api/endpoint', (req, res) => {
          return res.networkError('Failed to connect')
        })
      )

      await expect(apiService.getData()).rejects.toThrow()
    })
  })

  describe('createData', () => {
    it('应该成功创建数据', async () => {
      const newData = { name: 'new' }
      const createdData = { ...newData, id: 1 }

      server.use(
        http.post('/api/endpoint', (req, res, ctx) => {
          return res(
            ctx.status(201),
            ctx.json({ data: createdData })
          )
        })
      )

      const response = await apiService.createData(newData)

      expect(response.status).toBe(201)
      expect(response.data.id).toBe(1)
    })

    it('当参数无效时应该返回 400', async () => {
      server.use(
        http.post('/api/endpoint', (req, res, ctx) => {
          return res(
            ctx.status(400),
            ctx.json({ error: 'Invalid parameter' })
          )
        })
      )

      await expect(apiService.createData({})).rejects.toThrow()
    })
  })
})
```

```typescript
import { test, expect } from '@playwright/test'

test.describe('用户登录流程', () => {
```

### 6.1 覆盖率豁免

以下情况可以豁免测试覆盖：

1. **类型定义文件**（*.d.ts）
2. **配置文件**（*.config.*）
3. **入口文件**（main.tsx）
4. **第三方组件的包装层**（如 HOC）
5. **调试代码**（console.log、debugger 等）

## 九、最佳实践

### 9.1 查询元素优先级

| 优先级 | 查询方式 | 示例 |
|--------|---------|------|
| 1 | 可访问性角色 | `screen.getByRole('button')` |
| 2 | 可访问性标签 | `screen.getByLabelText('用户名')` |
| 3 | 测试 ID | `screen.getByTestId('submit-btn')` |
| 4 | 文本内容 | `screen.getByText('提交')` |
| 避免 | CSS 类名选择器 | `container.querySelector('.btn')` |

## 十一、常见问题

### Q1: 如何测试使用 Context 的组件？

使用 `renderHook` 从 `@testing-library/react`：

```typescript
import { renderHook } from '@testing-library/react'
import { useCustomHook } from './useCustomHook'

const { result } = renderHook(() => useCustomHook())
```

### Q2: 如何测试路由导航？

使用 MemoryRouter 或测试路由组件：

```typescript
import { BrowserRouter } from 'react-router-dom'
import { render, screen } from '@testing-library/react'

render(
  <BrowserRouter>
    <YourComponent />
  </BrowserRouter>
)
```

### Q3: 如何测试表单验证？

```typescript
import userEvent from '@testing-library/user-event'

it('应该验证必填字段', async () => {
  const user = userEvent.setup()
  render(<YourForm />)

  // 不填写直接提交
  await user.click(screen.getByRole('button', { name: /提交/i }))

  // 验证错误信息
  await expect(screen.getByText('请输入用户名')).toBeInTheDocument()
})
```

### 常用查询方式

```typescript
// 按角色查询
screen.getByRole('button')
screen.getAllByRole('link')

// 按文本查询
screen.getByText('提交')
screen.queryByText('取消')

// 按标签查询
screen.getByLabelText('用户名')
screen.queryByPlaceholderText('请输入密码')

// 按 testId 查询
screen.getByTestId('submit-btn')
screen.queryByTestId('loading-spinner')

// 按显示状态查询
screen.getAllByRole('listitem', { hidden: false })
```

### 常用断言

```typescript
// 元素存在
expect(element).toBeInTheDocument()
expect(element).toBeVisible()

// 元素不存在
expect(element).not.toBeInTheDocument()

// 文本内容
expect(element).toHaveText('预期文本')

// 属性
expect(element).toHaveAttribute('href', '/expected-path')
expect(element).toHaveClass('active')

// 表单值
expect(input).toHaveValue('test value')
```

```typescript
// Mock 模块
vi.mock('../services/api')

expect(mockFn).toHaveBeenCalledTimes(1)

```
