# Go 命名约定

## 包命名

- 全部小写，无下划线
- 简短且有意义
- 避免使用通用名（如 `util`, `common`）

## 文件命名

- 全部小写，使用下划线分隔
- 测试文件以 `_test.go` 结尾
- 平台相关文件使用后缀（如 `_linux.go`, `_windows.go`）

## 变量和函数命名

- 使用驼峰命名法
- 导出（公开）的首字母大写
- 未导出（私有）的首字母小写
- 接口名通常是方法名加 -er（如 Reader, Writer）

## 常量命名

- 使用驼峰命名法
- 导出常量首字母大写

## 错误处理

- 错误变量以 Err 开头（如 ErrNotFound）
- 使用 fmt.Errorf 添加上下文
- 避免使用 panic
